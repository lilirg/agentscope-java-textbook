# 课程教学计划（教师用）

> 本文件为教师提供完整的教学实施指导

---

## 一、课程基本信息

| 项目 | 内容 |
|------|------|
| **课程名称** | AI应用开发：从零到AgentScope-Java实战 |
| **学时** | 48学时（3学分） |
| **前置知识** | Java基础、数据结构 |
| **核心框架** | AgentScope-Java v0.2.0 |
| **教学模式** | 理论讲授（28学时） + 实战项目（18学时） + Capstone（2学时） |

---

## 二、教学目标

### 2.1 知识目标

- 掌握AI Agent核心概念（Perceptor-Planner-Executor-Memory）  
- 熟悉提示词工程与LLM API调用  
- 理解多Agent协作模式（Manager-Worker、Chain-of-Thought）  
- 掌握Agent安全防护与性能优化  
- 熟悉Agent应用部署与监控  

### 2.2 能力目标

- 能独立开发单Agent系统（工具调用+记忆）  
- 能设计多Agent协作系统（角色分工+通信协议）  
- 能实施安全防护（输入过滤+权限控制）  
- 能进行性能评测与优化  
- 能部署Agent应用到生产环境  

### 2.3 素质目标

- 树立工程化开发思维（测试→部署→监控）  
- 培养文档化习惯（ARCHITECTURE.md + AGENTS.md）  
- 增强安全意识（最小权限+审计日志）

## 四、详细教案（3.1-3.15）

### 3.1 第1章：认识AI Agent（2学时）

#### 教学目标
- 记忆：AI Agent核心特征与组件定义  
- 理解：Agent与传统AI系统的本质区别  

#### 教学内容
1. **什么是AI Agent**（40分钟）  
   - Agent = LLM + Planning + Tools + Memory  
   - 与传统AI系统的对比（规则系统/机器学习 vs 自主Agent）  
2. **Agent的典型应用场景**（40分钟）  
   - 个人助理、工作助手、专业领域案例  
   - 学科对齐：计算机专业毕业要求中的"AI应用开发"能力  

#### 教学形式
- 课堂讲授：30分钟  
- 案例分析：20分钟（对比传统AI vs Agent）  
- 课堂练习：10分钟（绘制Agent组件关系图）  

#### 作业
- 绘制Agent组件关系图（Mermaid流程图）  
- 简答：解释Agent的四个核心组件如何协同工作  

---

### 3.2 第2章：提示词工程（2学时）

#### 教学目标
- 应用：编写结构化提示词  
- 分析：不同LLM API的接口差异  

#### 教学内容
1. **提示词设计原则**（30分钟）  
   - CRISPE框架：Character、Role、Instance、Style、Persona、Environment  
   - 目标驱动 vs 问题驱动指令对比  
2. **LLM API调用基础**（50分钟）  
   - OpenAI API基础调用  
   - 阿里云通义千问API对比  
   - 错误处理与超时重试  

#### 教学形式
- 课堂讲授：20分钟  
- 实操演示：40分钟（调用OpenAI API）  
- 课堂练习：20分钟（编写CRISPE提示词）  

#### 作业
- 编写带参数验证的API调用代码  
- 对比OpenAI与通义千问API差异

### 3.3 第3章：API接入实践（4学时）

#### 教学目标
- 应用：配置多API接入系统  
- 评价：不同API的优劣与适用场景  

#### 教学内容
1. **本地模型部署**（60分钟）  
   - LLaMA.cpp + Qwen2模型  
   - REST封装  
2. **云端API安全实践**（60分钟）  
   - API密钥管理（环境变量/KMS）  
   - 访问控制与速率限制  

#### 教学形式
- 课堂讲授：20分钟  
- 实操演示：100分钟（本地模型部署 + API切换）  
- 课堂练习：40分钟（实现API切换逻辑）  

#### 作业
- 构建支持三重切换的天气Agent  
- 实现API切换逻辑（策略模式）

### 3.4 第4章：Agent架构（2学时）

#### 教学内容

   - Perceptor-Planner-Executor-Memory模块划分

   - ToolExecutor接口设计  
   - 异常处理：工具调用失败的降级策略  

#### 教学形式
- 课堂讲授：30分钟  
- 代码分析：60分钟（阅读AgentImplementation.java）  
- 课堂练习：30分钟（实现工具注册表）  

#### 作业

## 七、教学建议

### 7.1 针对零基础学生

- 第1-2章可适当延长，加强案例演示  
- 提供完整的代码模板（减少初学者负担）  
- 增加课堂练习时间（如：现场编写提示词）  

### 7.2 针对有经验学生

- 第3-4章可加快进度，增加难度（如：自定义Agent引擎）  
- 鼓励使用AgentScope-Java（而非自研）  
- 选题可更高阶（如：分布式多Agent系统）  

### 7.3 针对Capstone项目

- 提供3个标准选题（智能客服、知识管理、数据报告）  
- 每周安排1次项目指导（答疑 + 进度检查）  
- 采用"演示-评审-改进"循环（每周迭代）

## 八、常见问题

### Q1：学生Java基础薄弱怎么办？

**建议**：  
- 提前提供Java复习资料（并发、异常、设计模式）  
- 课堂补充Java核心概念（5分钟/次）  
- 使用简化版示例代码（减少语法复杂度）  

### Q2：AgentScope-Java版本更新怎么办？

**建议**：  
- 教材保持框架中立性（核心内容不依赖具体API）  
- 补充版本迁移指南（提供旧版→新版对照）  
- 鼓励学生对比不同版本API（拓展学习）  

### Q3：课堂时间不足怎么办？

**建议**：  
- 理论部分可压缩（每章保留核心内容）  
- 实操部分可调整为课后作业（提供详细指导）  
- Capstone项目可延长至2周（期末考试周前完成）
