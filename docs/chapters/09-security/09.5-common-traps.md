## 认证与授权陷阱

### 陷阱1：认证机制不安全

**问题描述**：
```
问题：AI应用认证机制存在安全风险
症状：
- API Key泄露
- 身份伪造
- 未授权访问

原因：
- API Key硬编码
- 未使用HTTPS
- 未实现Token刷新机制
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **API Key加密存储** | 使用AES加密存储API Key | 本地开发场景 |
| **Token认证** | 使用JWT实现Token认证 | Web应用场景 |
| **HTTPS强制** | 强制使用HTTPS通信 | 生产环境场景 |

**示例代码**：
```java
// API Key加密存储
public class SecureApiKeyStorage {
    private static final String AES_KEY = "your-256-bit-key"; // 实际使用时从环境变量读取
    
    public static String encrypt(String apiKey) throws Exception {
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(AES_KEY.getBytes(), "AES");
        
        GCMParameterSpec gcmParameterSpec = new GCMParameterSpec(128);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, gcmParameterSpec);
        
        byte[] encrypted = cipher.doFinal(apiKey.getBytes());
        return Base64.getEncoder().encodeToString(encrypted);
    }
    
    public static String decrypt(String encryptedApiKey) throws Exception {
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(AES_KEY.getBytes(), "AES");
        
        GCMParameterSpec gcmParameterSpec = new GCMParameterSpec(128);
        cipher.init(Cipher.DECRYPT_MODE, keySpec, gcmParameterSpec);
        
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(encryptedApiKey));
        return new String(decrypted);
    }
}

// Token认证
public class TokenAuthFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        String authHeader = request.getHeader("Authorization");
        
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            
            try {
                // 验证Token
                Jwts.parser().setSigningKey("your-secret-key").parseClaimsJws(token);
                filterChain.doFilter(request, response);
            } catch (Exception e) {
                response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
                response.getWriter().write("Invalid token");
                return;
            }
        } else {
            response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
            response.getWriter().write("Missing authorization header");
            return;
        }
    }
}

// HTTPS强制
@Configuration
public class HttpsConfig {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .requiresChannel(r -> r
                .requestMatchers(r -> r.getHeader("X-Forwarded-Proto") != null 
                    && r.getHeader("X-Forwarded-Proto").equals("https"))
                    .requiresSecure()
                .anyRequest().requiresInsecure()
            );
        return http.build();
    }
}
```

### 陷阱2：授权控制不严格

**问题描述**：
```
问题：AI应用授权控制不严格，导致权限泄露
症状：
- 用户访问越权数据
- 管理员权限被滥用
- 数据泄露风险

原因：
- 未实现RBAC（基于角色的访问控制）
- 权限检查未覆盖所有接口
- 权限配置不当
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **RBAC实现** | 基于角色的访问控制 | 通用场景 |
| **细粒度权限** | 权限控制到具体数据行 | 高安全场景 |
| **权限审计** | 定期审计权限配置 | 合规场景 |

**示例代码**：
```java
// RBAC实现
public class RbacService {
    private final Map<String, Set<String>> userRoles = new ConcurrentHashMap<>();
    private final Map<String, Set<String>> rolePermissions = new ConcurrentHashMap<>();
    
    public void assignRole(String userId, String role) {
        userRoles.computeIfAbsent(userId, k -> new HashSet<>()).add(role);
    }
    
    public void grantPermission(String role, String permission) {
        rolePermissions.computeIfAbsent(role, k -> new HashSet<>()).add(permission);
    }
    
    public boolean hasPermission(String userId, String permission) {
        Set<String> roles = userRoles.get(userId);
        if (roles == null) {
            return false;
        }
        
        for (String role : roles) {
            Set<String> permissions = rolePermissions.get(role);
            if (permissions != null && permissions.contains(permission)) {
                return true;
            }
        }
        
        return false;
    }
}

// 细粒度权限
public class FineGrainedPermissionFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) {
        String userId = request.getHeader("X-User-ID");
        String resourceId = request.getRequestURI();
        
        // 检查用户是否有权限访问该资源
        if (!hasPermission(userId, "resource:" + resourceId)) {
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.getWriter().write("Access denied");
            return;
        }
        
        filterChain.doFilter(request, response);
    }
    
    private boolean hasPermission(String userId, String permission) {
        // RBAC检查逻辑
        // 实际使用时需集成RBAC权限系统

    }
}
```

## 数据安全陷阱

### 陷阱3：敏感数据泄露

**问题描述**：
```
问题：AI应用处理过程中敏感数据泄露
症状：
- 用户隐私数据泄露
- API Key泄露
- 日志包含敏感信息

原因：
- 日志未脱敏
- 数据未加密
- 错误处理泄露信息
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **数据脱敏** | 日志中脱敏敏感数据 | 通用场景 |
| **数据加密** | 使用AES加密存储敏感数据 | 高安全场景 |
| **错误处理** | 错误信息不包含敏感细节 | 通用场景 |

**示例代码**：
```java
// 数据脱敏
public class DataMasker {
    public static String maskEmail(String email) {
        if (email == null || email.isEmpty()) {
            return email;
        }
        
        String[] parts = email.split("@");
        if (parts.length != 2) {
            return email;
        }
        
        String local = parts[0];
        String domain = parts[1];
        
        // 脱敏本地部分（保留首尾字符）
        String maskedLocal = local.charAt(0) + "***" + local.charAt(local.length() - 1);
        return maskedLocal + "@" + domain;
    }
    
    public static String maskApiKey(String apiKey) {
        if (apiKey == null || apiKey.length() < 8) {
            return "***";
        }
        
        return "***" + apiKey.substring(apiKey.length() - 4);
    }
}

// 数据加密
public class SecureDataStorage {
    private static final String AES_KEY = "your-256-bit-key";
    
    public static String encrypt(String data) throws Exception {
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(AES_KEY.getBytes(), "AES");
        
        GCMParameterSpec gcmParameterSpec = new GCMParameterSpec(128);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, gcmParameterSpec);
        
        byte[] encrypted = cipher.doFinal(data.getBytes());
        return Base64.getEncoder().encodeToString(encrypted);
    }
    
    public static String decrypt(String encryptedData) throws Exception {
        Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
        SecretKeySpec keySpec = new SecretKeySpec(AES_KEY.getBytes(), "AES");
        
        GCMParameterSpec gcmParameterSpec = new GCMParameterSpec(128);
        cipher.init(Cipher.DECRYPT_MODE, keySpec, gcmParameterSpec);
        
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(encryptedData));
        return new String(decrypted);
    }
}

// 错误处理
public class SafeErrorHandler {
    public static void handleException(Exception e) {
        logger.error("An error occurred", e);
        
        // 错误响应不包含敏感细节
        throw new RuntimeException("An error occurred. Please try again later.");
    }
}
```

### 陷阱4：SQL注入攻击

**问题描述**：
```
问题：AI应用存在SQL注入风险
症状：
- 数据库被恶意查询
- 数据被删除或篡改
- 数据泄露

原因：
- 使用字符串拼接SQL
- 未使用预编译语句
- 输入未验证
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **预编译语句** | 使用PreparedStatement | 通用场景 |
| **输入验证** | 验证输入格式与长度 | 通用场景 |
| **ORM框架** | 使用JPA/Hibernate | 复杂查询场景 |

**示例代码**：
```java
// 预编译语句
public class SafeDatabaseQuery {
    private final JdbcTemplate jdbcTemplate;
    
    public User findUserByEmail(String email) {
        String sql = "SELECT * FROM users WHERE email = ?";
        return jdbcTemplate.queryForObject(sql, new Object[]{email}, 
            new BeanPropertyRowMapper<>(User.class));
    }
    
    public List<Message> findMessagesByUserId(String userId) {
        String sql = "SELECT * FROM messages WHERE user_id = ? ORDER BY created_at DESC";
        return jdbcTemplate.query(sql, new Object[]{userId}, 
            new BeanPropertyRowMapper<>(Message.class));
    }
}

// 输入验证
public class InputValidator {
    public static void validateEmail(String email) {
        if (email == null || email.isEmpty()) {
            throw new IllegalArgumentException("Email cannot be empty");
        }
        
        if (email.length() > 256) {
            throw new IllegalArgumentException("Email too long");
        }
        
        if (!email.matches("^[A-Za-z0-9+_.-]+@([A-Za-z0-9.-]+\\.[A-Za-z]{2,})$")) {
            throw new IllegalArgumentException("Invalid email format");
        }
    }
    
    public static void validateApiKey(String apiKey) {
        if (apiKey == null || apiKey.isEmpty()) {
            throw new IllegalArgumentException("API Key cannot be empty");
        }
        
        if (apiKey.length() < 16) {
            throw new IllegalArgumentException("API Key too short");
        }
    }
}
```

### 陷阱5：模型幻觉问题

**问题描述**：
```
问题：LLM生成虚假信息（幻觉）
症状：
- 编造不存在的数据
- 错误引用文献
- 提供不准确的建议

原因：
- 模型训练数据局限
- 未检索真实数据
- 提示词设计不当
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **RAG增强** | 使用RAG检索真实数据 | 专业领域场景 |
| **置信度过滤** | 过滤低置信度回答 | 高安全场景 |
| **人工审核** | 关键结果人工审核 | 合规场景 |

**示例代码**：
```java
// RAG增强
public class RAGEnhancedAgent extends Agent {
    private final RAGRepository ragRepository;
    
    @Override
    public Response generate(String prompt) {
        // 检索相关数据
        List<String> retrievedDocs = ragRepository.retrieve(prompt, 3);
        
        // 构建增强提示词
        StringBuilder augmentedPrompt = new StringBuilder();
        augmentedPrompt.append("根据以下参考资料回答问题：\n\n");
        
        for (int i = 0; i < retrievedDocs.size(); i++) {
            augmentedPrompt.append("[").append(i + 1).append("] ").append(retrievedDocs.get(i)).append("\n");
        }
        
        augmentedPrompt.append("\n问题：").append(prompt).append("\n\n");
        augmentedPrompt.append("要求：\n");
        augmentedPrompt.append("1. 仅使用参考资料中的信息\n");
        augmentedPrompt.append("2. 如果参考资料不足，请说明\n");
        augmentedPrompt.append("3. 禁止编造信息\n");
        
        return super.generate(augmentedPrompt.toString());
    }
}
```

### 陷阱6：模型偏见问题

**问题描述**：
```
问题：LLM存在偏见（性别、种族、地域等）
症状：
- 生成偏见性内容
- 歪曲事实
- 不公平对待用户

原因：
- 训练数据偏见
- 提示词设计偏见
- 未进行偏见检测
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **偏见检测** | 使用偏见检测工具 | 开发/测试环境 |
| **中立提示词** | 设计中立的提示词 | 通用场景 |
| **人工审核** | 关键结果人工审核 | 合规场景 |

**示例代码**：
```java
// 偏见检测
public class BiasDetector {
    private static final List<String> biasedPatterns = Arrays.asList(
        "gender stereotypes",
        "racial bias",
        "地域歧视"
    );
    
    public boolean containsBias(String text) {
        for (String pattern : biasedPatterns) {
            if (text.toLowerCase().contains(pattern.toLowerCase())) {
                return true;
            }
        }
        
        return false;
    }
    
    public String removeBias(String text) {
        if (containsBias(text)) {
            // 返回中立版本
            return "I cannot provide a biased response. Please ask a neutral question.";
        }
        
        return text;
    }
}

// 中立提示词
public class NeutralPromptBuilder {
    public String buildNeutralPrompt(String userQuery) {
        StringBuilder prompt = new StringBuilder();
        
        prompt.append("请回答以下问题：\n\n");
        prompt.append(userQuery).append("\n\n");
        
        prompt.append("要求：\n");
        prompt.append("1. 保持中立客观\n");
        prompt.append("2. 避免偏见性内容\n");
        prompt.append("3. 尊重所有群体\n");
        
        return prompt.toString();
    }
}
```

### 陷阱7：安全日志不足

**问题描述**：
```
问题：安全调试缺乏足够日志信息
症状：
- 难以追踪安全事件
- 日志未记录敏感操作
- 安全事件复现困难
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **安全日志** | 记录所有安全相关操作 | 通用场景 |
| **Trace ID** | 为每个请求生成唯一Trace ID | 生产环境 |
| **审计日志** | 记录关键操作（如权限变更） | 合规场景 |

**示例代码**：
```java
// 安全日志
public class SecurityAuditLogger {
    private static final Logger logger = LoggerFactory.getLogger(SecurityAuditLogger.class);
    
    public void logSecurityEvent(String eventType, Map<String, String> details) {
        logger.info("[SECURITY] Event: {}, Details: {}", eventType, details);
    }
    
    public void logAuthenticationFailure(String userId, String reason) {
        logSecurityEvent("AUTH_FAILURE", Map.of("user_id", userId, "reason", reason));
    }
    
    public void logPermissionDenied(String userId, String resource) {
        logSecurityEvent("PERMISSION_DENIED", Map.of("user_id", userId, "resource", resource));
    }
}

// Trace ID
public class SecurityTrace {
    private static final ThreadLocal<String> traceId = ThreadLocal.withInitial(
        () -> UUID.randomUUID().toString().substring(0, 8)
    );
    
    public static String getTraceId() {
        return traceId.get();
    }
}

// 审计日志
public class AuditLogService {
    private final AuditLogRepository auditLogRepository;
    
    public void logAuditEvent(String userId, String action, String resource, 
                             Map<String, String> details) {
        AuditLog log = new AuditLog();
        log.setTraceId(SecurityTrace.getTraceId());
        log.setUserId(userId);
        log.setAction(action);
        log.setResource(resource);
        log.setDetails(details);
        log.setTimestamp(Instant.now());
        
        auditLogRepository.save(log);
    }
}
```
