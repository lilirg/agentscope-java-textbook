#### 2. 加密库（加密敏感数据）

| 库 | Maven坐标 | 说明 |
|----|-----------|------|
| **JCE**（Java Cryptography Extension） | JDK自带 | Java标准加密库（AES、RSA） |
| **Bouncy Castle** | `org.bouncycastle:bcprov-jdk15on:1.70` | 第三方加密库（增强JCE） |

**JCE使用示例**（AES加密）：
```java
import javax.crypto.Cipher;
import javax.crypto.KeyGenerator;
import javax.crypto.SecretKey;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;

public class AESCrypto {
    private static final String ALGORITHM = "AES";
    
    public static String encrypt(String data, String key) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.ENCRYPT_MODE, secretKey);
        byte[] encryptedData = cipher.doFinal(data.getBytes());
        return Base64.getEncoder().encodeToString(encryptedData);
    }
    
    public static String decrypt(String encryptedData, String key) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.DECRYPT_MODE, secretKey);
        byte[] decodedData = Base64.getDecoder().decode(encryptedData);
        byte[] decryptedData = cipher.doFinal(decodedData);
        return new String(decryptedData);
    }
}
```

#### 3. TLS/SSL（传输加密）

| 库 | Maven坐标 | 说明 |
|----|-----------|------|
| **Java SSL** | JDK自带 | Java标准TLS/SSL实现 |
| **Apache HttpClient** | `org.apache.httpcomponents:httpclient:4.5.14` | HTTP客户端（支持TLS） |

**TLS配置示例**：
```java
import javax.net.ssl.SSLContext;
import org.apache.http.impl.client.CloseableHttpClient;
import org.apache.http.impl.client.HttpClients;
import org.apache.http.conn.socket.ConnectionSocketFactory;
import org.apache.http.conn.socket.PlainConnectionSocketFactory;
import org.apache.http.conn.ssl.SSLConnectionSocketFactory;

// 1. 创建SSLContext（TLSv1.3）
SSLContext sslContext = SSLContext.getInstance("TLSv1.3");
sslContext.init(null, null, new java.security.SecureRandom());

// 2. 创建HttpClient（启用TLS）
ConnectionSocketFactory plainsf = PlainConnectionSocketFactory.getSocketFactory();
ConnectionSocketFactory sslsf = SSLConnectionSocketFactory.getSocketFactory();
Registry<ConnectionSocketFactory> registry = RegistryBuilder.<ConnectionSocketFactory>create()
    .register("http", plainsf)
    .register("https", sslsf)
    .build();

CloseableHttpClient httpClient = HttpClients.custom()
    .setSSLSocketFactory(sslsf)
    .build();
```

### 9.2.3 调试工具

#### 1. 日志审计（SLF4J + Logback）

| 库 | Maven坐标 | 说明 |
|----|-----------|------|
| SLF4J | `org.slf4j:slf4j-api:2.0.13` | 日志门面（统一日志API） |
| Logback | `ch.qos.logback:logback-classic:1.5.6` | 日志实现（SLF4J的默认实现） |

**日志配置示例**（`logback.xml`）：
```xml
<!-- src/main/resources/logback.xml -->
<configuration>
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>logs/security-audit.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>logs/security-audit.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    
    <root level="INFO">
        <appender-ref ref="FILE"/>
    </root>
</configuration>
```

#### 2. 异常检测（安全事件追踪）

| 工具 | 说明 |
|------|------|
| **自定义异常处理器** | 统一处理安全异常（API Key泄露、权限不足） |
| **告警系统** | 邮件/钉钉告警（安全事件发生时通知管理员） |

**自定义异常处理器示例**：
```java
@RestControllerAdvice
public class SecurityExceptionHandler {
    
    @ExceptionHandler(ApiKeyLeakException.class)
    public ResponseEntity<ErrorResponse> handleApiKeyLeak(ApiKeyLeakException e) {
        // 1. 记录日志
        log.error("API Key泄露风险: {}", e.getMessage());
        
        // 2. 发送告警（钉钉机器人）
        DingTalkNotifier.notify("API Key泄露风险", e.getMessage());
        
        // 3. 返回错误响应
        ErrorResponse error = new ErrorResponse("SECURITY_ERROR", "API Key泄露风险");
        return ResponseEntity.status(HttpStatus.FORBIDDEN).body(error);
    }
}
```

### 9.2.5 调试技巧

#### 1. 本地调试（开发环境）

| 技巧 | 说明 |
|------|------|
| **Mock API Key** | 使用假API Key调试（避免真实API调用） |
| **本地TLS** | 使用自签名证书测试TLS连接 |

**Mock API Key示例**：
```java
public class APIConfig {
    private final String apiKey;
    
    public APIConfig() {
        if (isDevEnvironment()) {
            this.apiKey = "sk-mock-key-for-dev"; // 开发环境Mock Key
        } else {
            this.apiKey = ConfigLoader.get("API_KEY"); // 生产环境真实Key
        }
    }
    
    private boolean isDevEnvironment() {
        return System.getenv("ENV") != null && 
               System.getenv("ENV").equals("dev");
    }
}
```

#### 2. 单元测试（安全功能验证）

| 工具 | Maven坐标 | 说明 |
|------|-----------|------|
| JUnit | `org.junit.jupiter:junit-jupiter:5.11.0` | 单元测试框架 |
| Mockito | `org.mockito:mockito-core:5.14.0` | Mock框架 |

**单元测试示例**（AES加密）：
```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AESCryptoTest {
    
    @Test
    public void testEncryptAndDecrypt() throws Exception {
        // 1. 准备
        AESCrypto crypto = new AESCrypto();
        String key = "1234567890123456"; // 16字节密钥
        String data = "Hello, World!";
        
        // 2. 执行
        String encrypted = crypto.encrypt(data, key);
        String decrypted = crypto.decrypt(encrypted, key);
        
        // 3. 验证
        assertNotEquals(data, encrypted); // 加密后不等于原文
        assertEquals(data, decrypted);    // 解密后等于原文
    }
}
```
