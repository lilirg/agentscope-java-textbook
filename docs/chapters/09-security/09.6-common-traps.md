## 9.5.1 API Key配置相关陷阱

### 9.5.1.1 陷阱1：API Key加载失败

**现象**：
```
Error: API_KEY not configured
```

**原因**：
- `.env`文件路径错误（未加载）
- 环境变量未设置
- 配置项名称拼写错误

**调试方法**：
```java
// 1. 检查.env文件是否加载
public class ConfigLoader {
    public static String get(String key) {
        Dotenv dotenv = Dotenv.configure().ignoreIfMissing().load();
        String value = dotenv.get(key);
        System.out.println("Config: " + key + " = " + value); // 调试输出
        return value;
    }
}

// 2. 检查环境变量
System.out.println("API_KEY from env: " + System.getenv("API_KEY"));
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **检查路径** | 确保`.env`文件在根目录 | 开发环境 |
| **环境变量** | 设置系统环境变量 | 生产环境 |
| **配置中心** | 使用Nacos/Apollo | 分布式系统 |

**示例**：
```java
public class APIConfig {
    public APIConfig() {
        // 1. 加载.env
        Dotenv dotenv = Dotenv.configure()
            .directory(".env") // 指定路径
            .ignoreIfMissing()
            .load();

        this.apiKey = dotenv.get("API_KEY");

        // 2. 检查API Key是否为空
        if (apiKey == null || apiKey.isEmpty()) {
            throw new IllegalStateException("API_KEY not configured");
        }
    }
}
```

### 9.5.1.2 陷阱2：API Key加密/解密失败

**现象**：
```
Error: javax.crypto.BadPaddingException: Given final block not properly padded
```

**原因**：
- 加密/解密密钥不一致
- 加密算法配置错误（如AES/CBC/PKCS5Padding）
- 数据格式错误（非Base64编码）

**调试方法**：
```java
// 1. 检查密钥长度（AES要求16/24/32字节）
byte[] keyBytes = encryptionKey.getBytes();
System.out.println("Key length: " + keyBytes.length + " bytes");

// 2. 检查加密数据格式
try {
    Base64.getDecoder().decode(encryptedData);
    System.out.println("Data is valid Base64");
} catch (IllegalArgumentException e) {
    System.out.println("Invalid Base64 data");
}
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **密钥一致性** | 加密/解密使用相同密钥 | 通用 |
| **算法配置** | 明确指定加密模式（AES/CBC/PKCS5Padding） | 加密敏感数据 |
| **数据校验** | 加密前校验数据格式 | 输入校验 |

**示例**：
```java
public class AESCrypto {
    private static final String ALGORITHM = "AES";
    private static final String TRANSFORMATION = "AES/CBC/PKCS5Padding"; // 明确指定

    public static String encrypt(String data, String key) throws Exception {
        if (key.length() != 32) {
            throw new IllegalArgumentException("Encryption key must be 32 bytes");
        }

        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        
        // 1. 生成IV（初始化向量）
        cipher.init(Cipher.ENCRYPT_MODE, secretKey);
        byte[] iv = cipher.getIV();

        // 2. 加密
        cipher.init(Cipher.ENCRYPT_MODE, secretKey, new IvParameterSpec(iv));
        byte[] encryptedData = cipher.doFinal(data.getBytes());

        // 3. 返回Base64编码（含IV）
        byte[] combined = new byte[iv.length + encryptedData.length];
        System.arraycopy(iv, 0, combined, 0, iv.length);
        System.arraycopy(encryptedData, 0, combined, iv.length, encryptedData.length);
        return Base64.getEncoder().encodeToString(combined);
    }

    public static String decrypt(String encryptedData, String key) throws Exception {
        if (key.length() != 32) {
            throw new IllegalArgumentException("Encryption key must be 32 bytes");
        }

        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);

        // 1. 解码Base64
        byte[] combined = Base64.getDecoder().decode(encryptedData);

        // 2. 提取IV
        byte[] iv = new byte[16];
        byte[] encryptedBytes = new byte[combined.length - 16];
        System.arraycopy(combined, 0, iv, 0, iv.length);
        System.arraycopy(combined, 16, encryptedBytes, 0, encryptedBytes.length);

        // 3. 解密
        cipher.init(Cipher.DECRYPT_MODE, secretKey, new IvParameterSpec(iv));
        byte[] decryptedData = cipher.doFinal(encryptedBytes);
        return new String(decryptedData);
    }
}
```

## 9.5.2 权限控制相关陷阱

### 9.5.2.1 陷阱3：RBAC权限校验绕过

**现象**：
```
Warning: User 'u123' bypassed permission check (admin role not assigned)
```

**原因**：
- `hasPermission()`方法未被调用
- `userRoles`映射未正确初始化
- 角色分配逻辑错误

**调试方法**：
```java
// 1. 检查权限校验是否被调用
public class RBAC {
    public boolean hasPermission(String userId, Permission permission) {
        log.debug("Checking permission: userId={}, permission={}", userId, permission);
        
        Role role = userRoles.get(userId);
        log.debug("User role: {}", role);

        if (role == null) {
            return false;
        }
        
        boolean hasPerm = role.getPermissions().contains(permission);
        log.debug("Permission check result: {}", hasPerm);
        
        return hasPerm;
    }
}

// 2. 检查角色分配
rbac.assignRole("u123", Role.ADMIN);
System.out.println("User u123 role: " + rbac.userRoles.get("u123"));
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **强制校验** | 在API拦截器中统一校验权限 | 通用 |
| **角色检查** | 在分配角色前检查角色有效性 | 角色管理 |
| **日志审计** | 记录权限变更日志 | 安全审计 |

**示例**（Spring Boot拦截器）：
```java
@Component
public class PermissionInterceptor implements HandlerInterceptor {
    
    @Autowired
    private RBAC rbac;

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        String userId = request.getHeader("X-User-ID");
        String method = request.getMethod();
        String path = request.getRequestURI();

        // 1. 检查权限（根据URL推断权限类型）
        Permission requiredPermission;
        if (method.equals("GET")) {
            requiredPermission = Permission.QUERY;
        } else if (method.equals("DELETE")) {
            requiredPermission = Permission.DELETE;
        } else {
            requiredPermission = Permission.CONFIG;
        }

        // 2. 权限校验
        if (!rbac.hasPermission(userId, requiredPermission)) {
            response.setStatus(HttpStatus.FORBIDDEN.value());
            response.getWriter().write("无权限访问");
            return false;
        }

        return true;
    }
}
```

### 9.5.2.2 陷阱4：ABAC属性获取失败

**现象**：
```
Error: NullPointerException at ABAC.checkAccess()
```

**原因**：
- `UserAttributes`未正确初始化
- 属性值为空（如department为空）
- 数据库查询失败

**调试方法**：
```java
public class ABAC {
    public boolean checkAccess(String userId, String resource, String action) {
        try {
            UserAttributes attributes = getUserAttributes(userId);
            System.out.println("User attributes: " + attributes); // 调试输出
            
            if (attributes == null) {
                throw new NullPointerException("UserAttributes is null for userId: " + userId);
            }
            
            if (attributes.getDepartment() == null) {
                throw new NullPointerException("Department is null for userId: " + userId);
            }

            return (attributes.getDepartment().equals("IT") || 
                    attributes.getDepartment().equals("HR")) &&
                   isWorkday() &&
                   isInternalNetwork(attributes.getLocation());

        } catch (Exception e) {
            log.error("ABAC check failed for userId: {}", userId, e);
            return false;
        }
    }

    private UserAttributes getUserAttributes(String userId) {
        // 从数据库/LDAP获取用户属性
        // 实际使用时需集成具体数据库
        return new UserAttributes(userId, "IT", "Beijing", LocalDateTime.now());
    }
}
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **空值检查** | 对所有属性进行空值检查 | 通用 |
| **默认值** | 为空时使用默认属性值 | 用户属性缺失 |
| **异常处理** | 捕获并记录异常 | 安全审计 |

## 9.5.3 数据安全相关陷阱

### 9.5.3.1 陷阱5：TLS证书验证失败

**现象**：
```
Error: javax.net.ssl.SSLHandshakeException: sun.security.validator.ValidatorException: PKIX path building failed
```

**原因**：
- 证书过期
- 证书链不完整
- 证书颁发机构不受信任

**调试方法**：
```bash
## 1. 检查证书有效期
openssl x509 -in server.crt -noout -dates

## 2. 检查证书链
openssl verify -show_chain server.crt

## 3. 检查证书颁发机构
openssl x509 -in server.crt -noout -issuer
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **自签名证书** | 开发环境使用自签名证书 | 开发/测试 |
| **CA证书** | 生产环境使用CA证书 | 生产环境 |
| **忽略验证** | 临时跳过验证（不推荐） | 调试 |

**示例**（开发环境临时忽略验证）：
```java
import javax.net.ssl.SSLContext;
import javax.net.ssl.TrustManager;
import javax.net.ssl.X509TrustManager;
import java.security.cert.X509Certificate;

public class TLSClient {
    public static HttpClient createUnsafeHttpClient() {
        try {
            // 1. 创建信任所有证书的TrustManager
            TrustManager[] trustAllCerts = new TrustManager[]{
                new X509TrustManager() {
                    public X509Certificate[] getAcceptedIssuers() {
                        return new X509Certificate[0];
                    }
                    public void checkServerTrusted(X509Certificate[] certs, String authType) {}
                    public void checkClientTrusted(X509Certificate[] certs, String authType) {}
                }
            };

            // 2. 创建SSLContext
            SSLContext sslContext = SSLContext.getInstance("TLS");
            sslContext.init(null, trustAllCerts, new java.security.SecureRandom());

            // 3. 创建HttpClient
            return HttpClient.newBuilder()
                .sslContext(sslContext)
                .build();

        } catch (Exception e) {
            throw new RuntimeException("Failed to create unsafe HTTP client", e);
        }
    }
}
```

## 9.5.4 安全审计相关陷阱

### 9.5.4.1 陷阱6：日志记录不完整

**现象**：
```
Warning: Audit log incomplete (missing user ID)
```

**原因**：
- 日志模板未包含必要字段
- 拦截器未捕获所有请求
- 异常未记录到审计日志

**调试方法**：
```java
// 1. 检查日志模板
logger.info("API调用: userId={}, api={}, parameters={}", userId, api, parameters);

// 2. 检查拦截器
@Component
public class AuditInterceptor implements HandlerInterceptor {
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) {
        if (ex != null) {
            auditService.logException(request.getHeader("X-User-ID"), request.getRequestURI(), ex);
        }
    }
}
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **完整模板** | 日志包含所有必要字段（用户、API、参数、时间） | 通用 |
| **异常记录** | 捕获并记录所有异常 | 安全审计 |
| **日志校验** | 定期检查日志完整性 | 运维 |

## 9.5.5 通用调试技巧

### 9.5.5.1 单元测试（JUnit + Mockito）

**测试示例**（AES加密）：
```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class AESCryptoTest {
    
    @Test
    public void testEncryptAndDecrypt() throws Exception {
        // 1. 准备
        String key = "1234567890123456"; // 16字节密钥
        String data = "Hello, World!";
        
        // 2. 执行
        String encrypted = AESCrypto.encrypt(data, key);
        String decrypted = AESCrypto.decrypt(encrypted, key);
        
        // 3. 验证
        assertNotEquals(data, encrypted); // 加密后不等于原文
        assertEquals(data, decrypted);    // 解密后等于原文
    }
    
    @Test
    public void testInvalidKeyLength() {
        // 1. 准备
        String key = "123"; // 无效密钥长度
        String data = "Hello, World!";
        
        // 2. 执行 + 3. 验证
        assertThrows(IllegalArgumentException.class, () -> {
            AESCrypto.encrypt(data, key);
        });
    }
}
```

### 9.5.5.2 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class SecurityMonitor {
    private static final Logger logger = LoggerFactory.getLogger(SecurityMonitor.class);
    
    public void monitorAPICall(String userId) {
        logger.debug("Monitoring API call: userId={}", userId);
        
        apiCallCount.merge(userId, 1, Integer::sum);
        
        if (apiCallCount.get(userId) > 100) {
            logger.error("API调用异常: userId={}, count={}", userId, apiCallCount.get(userId));
            suspiciousUsers.add(userId);
        }
    }
}
```
