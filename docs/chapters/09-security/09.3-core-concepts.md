### 9.3.1 API Key安全

#### 1. 环境变量（Environment Variables）

**原理**：将敏感配置存储在环境变量中，避免硬编码到代码中。

**优势**：
- 代码不包含密钥（Git提交安全）
- 不同环境使用不同密钥（dev/test/prod）

**实现示例**：
```java
public class APIConfig {
    private final String apiKey;
    
    public APIConfig() {
        // 1. 优先从环境变量读取
        this.apiKey = System.getenv("API_KEY");
        
        // 2. 如果环境变量不存在，从.env文件读取
        if (apiKey == null) {
            this.apiKey = ConfigLoader.get("API_KEY");
        }
        
        // 3. 如果都不存在，抛出异常
        if (apiKey == null || apiKey.isEmpty()) {
            throw new IllegalStateException("API_KEY not configured");
        }
    }
}
```

**配置方式**：
- **Linux/macOS**：
  ```bash
  export API_KEY=your_api_key_here
  java -jar app.jar
  ```
- **Windows**：
  ```cmd
  set API_KEY=your_api_key_here
  java -jar app.jar
  ```

#### 2. 加密存储（AES加密）

**原理**：使用AES加密算法对密钥进行加密存储，解密时使用主密钥解密。

**加密流程**：
```
原始密钥 → AES加密 → 加密密钥（存储在配置文件）
配置文件 → AES解密 → 原始密钥（运行时使用）
```

**实现示例**：
```java
public class EncryptedAPIKey {
    private static final String ALGORITHM = "AES";
    private static final String TRANSFORMATION = "AES";
    
    public static String encrypt(String apiKey, String encryptionKey) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(encryptionKey.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        cipher.init(Cipher.ENCRYPT_MODE, secretKey);
        byte[] encryptedData = cipher.doFinal(apiKey.getBytes());
        return Base64.getEncoder().encodeToString(encryptedData);
    }
    
    public static String decrypt(String encryptedApiKey, String encryptionKey) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(encryptionKey.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        cipher.init(Cipher.DECRYPT_MODE, secretKey);
        byte[] decodedData = Base64.getDecoder().decode(encryptedApiKey);
        byte[] decryptedData = cipher.doFinal(decodedData);
        return new String(decryptedData);
    }
}
```

**使用方式**：
1. **初始化**（首次配置）：
   ```bash
   # 生成32字节主密钥（用于加密/解密）
   openssl rand -base64 32 > encryption_key.txt
   ```
   
2. **加密原始密钥**：
   ```java
   String originalApiKey = System.getenv("API_KEY"); // 从环境变量读取
   String encryptionKey = Files.readString(Path.of("encryption_key.txt")).trim();
   String encryptedApiKey = EncryptedAPIKey.encrypt(originalApiKey, encryptionKey);
   System.out.println("加密密钥：" + encryptedApiKey);
   ```

3. **解密运行时密钥**：
   ```java
   String encryptedApiKey = System.getenv("ENCRYPTED_API_KEY");
   String encryptionKey = Files.readString(Path.of("encryption_key.txt")).trim();
   String apiKey = EncryptedAPIKey.decrypt(encryptedApiKey, encryptionKey);
   ```

#### 3. 密钥轮换（Key Rotation）

**原理**：定期更换API密钥，降低密钥泄露风险。

**实现策略**：
- **自动轮换**：每30天自动生成新密钥
- **双密钥机制**：同时使用两个密钥（主密钥 + 备用密钥）

**实现示例**：
```java
public class KeyRotation {
    private final String apiKey;
    private final String backupApiKey;
    private final long apiKeyCreatedAt;
    
    public KeyRotation() {
        this.apiKey = ConfigLoader.get("API_KEY");
        this.backupApiKey = ConfigLoader.get("BACKUP_API_KEY");
        this.apiKeyCreatedAt = Long.parseLong(ConfigLoader.get("API_KEY_CREATED_AT"));
    }
    
    public String getApiKey() {
        // 1. 检查密钥有效期（30天）
        long now = System.currentTimeMillis();
        long keyAgeDays = (now - apiKeyCreatedAt) / (1000 * 60 * 60 * 24);
        
        if (keyAgeDays > 30) {
            // 2. 密钥过期，使用备用密钥
            log.warn("API密钥已过期，使用备用密钥");
            return backupApiKey;
        }
        
        // 3. 密钥有效，使用主密钥
        return apiKey;
    }
}
```

### 9.3.2 权限控制

#### 1. RBAC（基于角色的访问控制）

**原理**：用户分配角色，角色分配权限。

**权限模型**：
```
用户 → 角色 → 权限
用户A → 管理员 → [查询、删除、配置]
用户B → 普通用户 → [查询]
```

**实现示例**：
```java
public enum Permission {
    QUERY,  // 查询权限
    DELETE, // 删除权限
    CONFIG  // 配置权限
}

public enum Role {
    ADMIN(Arrays.asList(Permission.QUERY, Permission.DELETE, Permission.CONFIG)),
    USER(Arrays.asList(Permission.QUERY));
    
    private final List<Permission> permissions;
    
    Role(List<Permission> permissions) {
        this.permissions = permissions;
    }
    
    public List<Permission> getPermissions() {
        return permissions;
    }
}

public class RBAC {
    private final Map<String, Role> userRoles = new HashMap<>();
    
    public void assignRole(String userId, Role role) {
        userRoles.put(userId, role);
    }
    
    public boolean hasPermission(String userId, Permission permission) {
        Role role = userRoles.get(userId);
        if (role == null) {
            return false;
        }
        return role.getPermissions().contains(permission);
    }
}
```

**使用方式**：
```java
RBAC rbac = new RBAC();
rbac.assignRole("userA", Role.ADMIN);
rbac.assignRole("userB", Role.USER);

// 1. 检查权限
boolean canDelete = rbac.hasPermission("userA", Permission.DELETE);
System.out.println("用户A有删除权限：" + canDelete); // true

// 2. API接口权限校验
@RestController
public class APIController {
    @Autowired
    private RBAC rbac;
    
    @PostMapping("/delete")
    public ResponseEntity<String> delete(@RequestHeader("X-User-ID") String userId) {
        if (!rbac.hasPermission(userId, Permission.DELETE)) {
            return ResponseEntity.status(HttpStatus.FORBIDDEN).body("无删除权限");
        }
        // 执行删除逻辑
        return ResponseEntity.ok("删除成功");
    }
}
```

#### 2. ABAC（基于属性的访问控制）

**原理**：根据用户属性（部门、时间、地理位置）动态判断权限。

**权限模型**：
```
用户（部门=IT，时间=工作日，地点=办公室） → 允许访问
用户（部门=HR，时间=休息日，地点=外网） → 拒绝访问
```

**实现示例**：
```java
public class ABAC {
    
    public boolean checkAccess(String userId, String resource, String action) {
        // 1. 获取用户属性
        UserAttributes attributes = getUserAttributes(userId);
        
        // 2. 判断权限
        // 限制条件：
        // - 部门必须是IT或HR
        // - 时间必须是工作日（周一至周五）
        // - 地理位置必须是公司内网
        return (attributes.getDepartment().equals("IT") || 
                attributes.getDepartment().equals("HR")) &&
               isWorkday() &&
               isInternalNetwork(attributes.getLocation());
    }
    
    private UserAttributes getUserAttributes(String userId) {
        // 从数据库/LDAP获取用户属性
        return new UserAttributes(userId, "IT", "Beijing", LocalDateTime.now());
    }
    
    private boolean isWorkday() {
        DayOfWeek day = LocalDateTime.now().getDayOfWeek();
        return day != DayOfWeek.SATURDAY && day != DayOfWeek.SUNDAY;
    }
    
    private boolean isInternalNetwork(String location) {
        // 检查IP地址是否在内网范围内
        return location.equals("Beijing") || location.equals("Shanghai");
    }
}
```

---

#### 3. RBAC vs ABAC对比

| 特性 | RBAC | ABAC |
|------|------|------|
| **权限分配** | 静态（用户→角色→权限） | 动态（用户属性→权限） |
| **适用场景** | 简单场景（如客服系统角色） | 复杂场景（如医疗系统权限） |
| **实现复杂度** | 低 | 高 |
| **性能** | 快 | 慢（需动态计算） |

**选型建议**：
- **RBAC**：权限固定、角色清晰的系统（如客服、管理后台）
- **ABAC**：权限动态变化的系统（如医疗、金融）

### 9.3.3 数据安全

#### 1. 加密存储（静态数据）

**原理**：对数据库中的敏感数据进行加密存储，防止数据库泄露后数据暴露。

**加密字段**：
- 手机号
- 地址
- 身份证号
- 医疗记录

**实现示例**：
```java
@Entity
public class User {
    @Id
    private String id;
    
    private String name;
    
    // 加密存储手机号
    @Column(length = 255)
    private String encryptedPhone;
    
    // 加密存储地址
    @Column(length = 512)
    private String encryptedAddress;
    
    // 临时字段（运行时解密）
    @Transient
    private String decryptedPhone;
    
    @Transient
    private String decryptedAddress;
    
    // 获取解密后数据

        if (decryptedPhone == null && encryptedPhone != null) {
            try {
                decryptedPhone = AESCrypto.decrypt(encryptedPhone, encryptionKey);
            } catch (Exception e) {
                throw new RuntimeException("解密失败", e);
            }
        }
        return decryptedPhone;
    }
}
```

#### 2. TLS传输（动态数据）

**原理**：使用TLS协议加密网络传输，防止中间人攻击。

**配置示例**（Spring Boot）：
```yaml
# application.yml
server:
  ssl:
    enabled: true
    key-store: classpath:keystore.p12
    key-store-password: your-password
    key-store-type: PKCS12
    key-alias: tomcat
```

**客户端TLS配置**：
```java
RestTemplate restTemplate = new RestTemplate();
HttpComponentsClientHttpRequestFactory factory = new HttpComponentsClientHttpRequestFactory();
factory.setConnectTimeout(5000);
factory.setReadTimeout(5000);
restTemplate.setRequestFactory(new HttpComponentsClientHttpRequestFactory(factory));
```

### 9.3.4 安全审计

#### 1. 日志记录（SLF4J）

**审计日志内容**：
- API调用记录（时间、用户、API、参数）
- 权限变更记录（角色分配、权限修改）
- 数据访问记录（敏感数据查询）

**实现示例**：
```java
@Service
public class AuditService {
    private static final Logger logger = LoggerFactory.getLogger(AuditService.class);
    
    public void logAPICall(String userId, String api, String parameters) {
        logger.info("API调用: userId={}, api={}, parameters={}", userId, api, parameters);
    }
    
    public void logPermissionChange(String userId, String action, String details) {
        logger.info("权限变更: userId={}, action={}, details={}", userId, action, details);
    }
    
    public void logDataAccess(String userId, String resource, String action) {
        logger.info("数据访问: userId={}, resource={}, action={}", userId, resource, action);
    }
}
```

#### 2. 异常检测（安全事件追踪）

**检测逻辑**：
- 单位时间API调用次数异常（可能被滥用）
- 非工作时间访问敏感数据（可能被窃取）
- 权限变更异常（可能被篡改）

**实现示例**：
```java
@Service
public class SecurityMonitor {
    private final Map<String, Integer> apiCallCount = new ConcurrentHashMap<>();
    private final Set<String> suspiciousUsers = new ConcurrentHashMapKeySet<>();
    
    public void monitorAPICall(String userId) {
        // 1. 统计API调用次数
        apiCallCount.merge(userId, 1, Integer::sum);
        
        // 2. 检测异常（1分钟超过100次）
        if (apiCallCount.get(userId) > 100) {
            log.error("API调用异常: userId={}, count={}", userId, apiCallCount.get(userId));
            suspiciousUsers.add(userId);
        }
    }
    
    public boolean isSuspicious(String userId) {
        return suspiciousUsers.contains(userId);
    }
}
```
