## 为什么需要本节技术？（现实场景引入）

> **学习目标**：理解安全防护的必要性

### 场景1：API Key硬编码导致的安全泄露

| 问题 | 解决方案 |
|------|---------|
| 密钥暴露在代码中 | 环境变量 + .env + 加密存储 |

### 场景2：用户权限管理缺失

| 问题 | 解决方案 |
|------|---------|
| 所有用户拥有相同权限 | RBAC/ABAC权限控制 |

### 场景3：敏感数据未加密

| 问题 | 解决方案 |
|------|---------|
| 明文存储/传输 | 加密存储（AES） + TLS传输 |

### 小结与思考题

- 问题1：为什么硬编码API Key是高危行为？  
- 问题2：RBAC与ABAC的区别是什么？  
- 问题3：敏感数据加密的两种场景是什么？

## 核心概念

> **学习目标**：掌握安全与权限控制的核心原理

### API Key安全

| 特性 | 实现要点 |
|------|---------|
| 环境变量 | `System.getenv("API_KEY")` |
| 加密存储 | AES加密 + Base64编码 |
| 密钥轮换 | 30天有效期 + 备用密钥 |

### 权限控制

| 特性 | 实现要点 |
|------|---------|
| RBAC | 用户→角色→权限 |
| ABAC | 用户属性→权限 |
| 拦截器 | Spring Interceptor统一校验 |

### 数据安全

| 特性 | 实现要点 |
|------|---------|
| 加密存储 | AES加密（静态数据） |
| TLS传输 | TLSv1.3（动态数据） |
| 敏感字段 | 手机号、地址、身份证号加密 |

### 安全审计

| 特性 | 实现要点 |
|------|---------|
| 日志记录 | SLF4J + Logback |
| 异常检测 | 单位时间API调用统计 |
| 告警系统 | 钉钉/邮件告警 |

## 常见陷阱与调试技巧

> **学习目标**：掌握典型问题与解决方案

### API Key配置

| 问题 | 解决方案 |
|------|---------|
| API Key加载失败 | 检查环境变量、.env路径 |
| 加密/解密失败 | 检查密钥一致性、算法配置 |

### 权限控制

| 问题 | 解决方案 |
|------|---------|
| 权限校验绕过 | 拦截器统一校验 + 空值检查 |
| ABAC属性获取失败 | 空值检查 + 默认值 + 异常处理 |

### 数据安全

| 问题 | 解决方案 |
|------|---------|
| TLS证书验证失败 | 检查证书有效期、证书链 |
