### 9.6.2 代码改错（每题15分，共30分）

#### 1. API Key配置改错

```java
// ❌ 错误代码（APIConfig.java）
public class APIConfig {
    private final String apiKey = System.getenv("API_KEY"); // 从环境变量读取

    public String getApiKey() {
        return apiKey;
    }
}
```

**问题**：  
1. 硬编码密钥暴露在代码中  
2. 无法支持密钥轮换

**正确代码**：
```java
public class APIConfig {
    private final String apiKey;

    public APIConfig() {
        // 1. 从环境变量读取
        this.apiKey = System.getenv("API_KEY");
        
        // 2. 如果环境变量不存在，从.env文件读取
        if (apiKey == null) {
            this.apiKey = ConfigLoader.get("API_KEY");
        }
        
        // 3. 如果都不存在，抛出异常
        if (apiKey == null || apiKey.isEmpty()) {
            throw new IllegalStateException("API_KEY not configured");
        }
    }

    public String getApiKey() {
        return apiKey;
    }
}
```

#### 2. 权限控制改错

```java
// ❌ 错误代码（RBAC.java）
public class RBAC {
    private final Map<String, Role> userRoles = new HashMap<>();

    public boolean hasPermission(String userId, Permission permission) {
        Role role = userRoles.get(userId); // ❌ 未检查null
        return role.getPermissions().contains(permission); // ❌ NPE
    }
}
```

**问题**：  
1. 未检查`role`是否为null  
2. 可能导致`NullPointerException`

**正确代码**：
```java
public class RBAC {
    private final Map<String, Role> userRoles = new ConcurrentHashMap<>();

    public boolean hasPermission(String userId, Permission permission) {
        Role role = userRoles.get(userId);
        if (role == null) {
            return false; // ✅ 明确返回false
        }
        return role.getPermissions().contains(permission);
    }
}
```

### 9.6.3 小型设计（30分）

#### 设计任务：实现ABAC权限控制

**需求**：  
设计一个`ABAC`类，实现基于属性的权限控制。

**功能要求**：  
1. 用户属性包含部门、地点、时间  
2. 限制条件：部门=IT/HR、时间=工作日、地点=内网  
3. 方法签名：`boolean checkAccess(String userId, String resource, String action)`

**参考答案**：
```java
public class ABAC {
    
    public boolean checkAccess(String userId, String resource, String action) {
        // 1. 获取用户属性
        UserAttributes attributes = getUserAttributes(userId);
        
        // 2. 判断权限
        // 限制条件：
        // - 部门必须是IT或HR
        // - 时间必须是工作日（周一至周五）
        // - 地理位置必须是公司内网
        return (attributes.getDepartment().equals("IT") || 
                attributes.getDepartment().equals("HR")) &&
               isWorkday() &&
               isInternalNetwork(attributes.getLocation());
    }

    private UserAttributes getUserAttributes(String userId) {
        // 从数据库/LDAP获取用户属性
        return new UserAttributes(userId, "IT", "Beijing", LocalDateTime.now());
    }

    private boolean isWorkday() {
        DayOfWeek day = LocalDateTime.now().getDayOfWeek();
        return day != DayOfWeek.SATURDAY && day != DayOfWeek.SUNDAY;
    }

    private boolean isInternalNetwork(String location) {
        // 检查IP地址是否在内网范围内
        return location.equals("Beijing") || location.equals("Shanghai");
    }
}
```

### 9.6.4 拓展挑战（20分）

#### 挑战1：API Key加密存储

**背景**：  
当前API Key使用明文存储，存在泄露风险。

**问题**：  
1. 如何使用AES加密API Key？  
2. 如何在运行时解密API Key？  
3. 如何实现密钥轮换（30天自动更换）？

**参考答案**：

```java
public class EncryptedAPIKey {
    private static final String ALGORITHM = "AES";
    private static final String TRANSFORMATION = "AES/CBC/PKCS5Padding";

    public static String encrypt(String apiKey, String encryptionKey) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(encryptionKey.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        cipher.init(Cipher.ENCRYPT_MODE, secretKey);
        byte[] iv = cipher.getIV();
        
        cipher.init(Cipher.ENCRYPT_MODE, secretKey, new IvParameterSpec(iv));
        byte[] encryptedData = cipher.doFinal(apiKey.getBytes());
        
        byte[] combined = new byte[iv.length + encryptedData.length];
        System.arraycopy(iv, 0, combined, 0, iv.length);
        System.arraycopy(encryptedData, 0, combined, iv.length, encryptedData.length);
        
        return Base64.getEncoder().encodeToString(combined);
    }

    public static String decrypt(String encryptedApiKey, String encryptionKey) throws Exception {
        SecretKeySpec secretKey = new SecretKeySpec(encryptionKey.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        
        byte[] combined = Base64.getDecoder().decode(encryptedApiKey);
        byte[] iv = new byte[16];
        byte[] encryptedBytes = new byte[combined.length - 16];
        System.arraycopy(combined, 0, iv, 0, iv.length);
        System.arraycopy(combined, 16, encryptedBytes, 0, encryptedBytes.length);
        
        cipher.init(Cipher.DECRYPT_MODE, secretKey, new IvParameterSpec(iv));
        byte[] decryptedData = cipher.doFinal(encryptedBytes);
        return new String(decryptedData);
    }
}
```

**密钥轮换实现**：
```java
public class KeyRotation {
    private final String apiKey;
    private final String backupApiKey;
    private final long apiKeyCreatedAt;

    public KeyRotation() {
        this.apiKey = ConfigLoader.get("ENCRYPTED_API_KEY");
        this.backupApiKey = ConfigLoader.get("BACKUP_ENCRYPTED_API_KEY");
        this.apiKeyCreatedAt = Long.parseLong(ConfigLoader.get("API_KEY_CREATED_AT"));
    }

    public String getApiKey(String encryptionKey) {
        long now = System.currentTimeMillis();
        long keyAgeDays = (now - apiKeyCreatedAt) / (1000 * 60 * 60 * 24);

        if (keyAgeDays > 30) {
            return AESCrypto.decrypt(backupApiKey, encryptionKey); // 使用备用密钥
        }

        return AESCrypto.decrypt(apiKey, encryptionKey); // 使用主密钥
    }
}
```

#### 挑战2：安全审计日志分析

**背景**：  
当前系统存在API调用异常，需要分析日志。

**问题**：  
1. 如何统计单位时间内的API调用次数？  
2. 如何识别异常API调用（如1分钟超过100次）？  
3. 如何实现告警（邮件/钉钉）？

**参考答案**：

```java
@Service
public class AuditAnalysisService {
    private final Map<String, Integer> apiCallCount = new ConcurrentHashMap<>();
    private final Map<String, Long> lastResetTime = new ConcurrentHashMap<>();
    private static final long RESET_INTERVAL = 60_000; // 1分钟

    public void logAPICall(String userId) {
        long now = System.currentTimeMillis();
        
        // 1. 重置计数器（每1分钟）
        lastResetTime.computeIfAbsent(userId, k -> now);
        if (now - lastResetTime.get(userId) > RESET_INTERVAL) {
            apiCallCount.put(userId, 0);
            lastResetTime.put(userId, now);
        }

        // 2. 统计API调用次数
        apiCallCount.merge(userId, 1, Integer::sum);

        // 3. 检测异常（1分钟超过100次）
        if (apiCallCount.get(userId) > 100) {
            log.error("API调用异常: userId={}, count={}", userId, apiCallCount.get(userId));
            sendAlert(userId, "API调用异常", "1分钟内调用次数超过100次");
        }
    }

    private void sendAlert(String userId, String title, String message) {
        // 钉钉告警
        DingTalkNotifier.notify(title, message);
        
        // 邮件告警（可选）
        // EmailNotifier.notify(userId, title, message);
    }
}
```
