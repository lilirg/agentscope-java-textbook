## 6.5.1 图像处理陷阱

### 陷阱1：图像格式不兼容

**问题描述**：
```
问题：上传的图像格式不被模型支持
症状：
- API返回`Invalid Image Format`错误
- CLIP模型无法加载图像
- 图像预处理失败

原因：
- 图像格式非JPEG/PNG（如WebP、BMP）
- 图像分辨率超出限制
- 图像文件损坏
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **格式转换** | 使用OpenCV转换为JPEG/PNG | 通用场景 |
| **尺寸裁剪** | 调整图像尺寸（如224x224） | 深度学习模型 |
| **格式验证** | 上传前验证图像格式与尺寸 | 前端/后端 |

**示例代码**：
```java
// 使用OpenCV进行格式转换
public BufferedImage convertToJPEG(String filePath) {
    Mat mat = Imgcodecs.imread(filePath);
    if (mat.empty()) {
        throw new IllegalArgumentException("Failed to load image: " + filePath);
    }
    
    // 转换为RGB格式
    Mat rgbMat = new Mat();
    Imgproc.cvtColor(mat, rgbMat, Imgproc.COLOR_BGR2RGB);
    
    // 转换为BufferedImage
    int width = rgbMat.cols();
    int height = rgbMat.rows();
    byte[] data = new byte[width * height * 3];
    rgbMat.get(0, 0, data);
    
    BufferedImage image = new BufferedImage(width, height, BufferedImage.TYPE_INT_RGB);
    WritableRaster raster = image.getRaster();
    raster.setDataElements(0, 0, width, height, data);
    
    return image;
}

// 图像尺寸验证
public void validateImage(BufferedImage image) {
    int width = image.getWidth();
    int height = image.getHeight();
    
    // 检查尺寸（如CLIP模型要求224x224）
    if (width != 224 || height != 224) {
        throw new IllegalArgumentException("Image size must be 224x224, but got " + 
            width + "x" + height);
    }
}
```

### 陷阱2：图像向量化耗时长

**问题描述**：
```
问题：CLIP模型向量化大量图像时耗时长
症状：
- 响应时间长（>5秒）
- CPU/GPU占用率高
- 请求堆积

原因：
- 模型未启用GPU加速
- 批处理大小过大
- 未缓存向量结果
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **启用GPU** | 使用CUDA加速向量化 | GPU可用场景 |
| **批量处理** | 分批处理图像（如10张/批） | 高并发场景 |
| **向量缓存** | 缓存向量化结果（Redis） | 重复查询场景 |

**示例代码**：
```java
// 批量处理
public List<Embedding> batchEncodeImages(List<BufferedImage> images, int batchSize) {
    List<Embedding> embeddings = new ArrayList<>();
    
    for (int i = 0; i < images.size(); i += batchSize) {
        List<BufferedImage> batch = images.subList(i, 
            Math.min(i + batchSize, images.size()));
        
        // 批量向量化
        embeddings.addAll(clipModel.encode(batch));
    }
    
    return embeddings;
}

// 向量缓存
public Embedding getCachedEmbedding(String imageId) {
    String cacheKey = "image:embed:" + imageId;
    String json = redisTemplate.opsForValue().get(cacheKey);
    
    if (json != null) {
        logger.info("Cache hit for image: {}", imageId);
        return Embedding.fromJson(json);
    }
    
    // 缓存未命中，向量化
    BufferedImage image = loadImage(imageId);
    Embedding embedding = clipModel.encode(image);
    
    // 缓存结果（TTL: 1天）
    redisTemplate.opsForValue().set(cacheKey, embedding.toJson(), 
        Duration.ofDays(1));
    
    return embedding;
}
```

## 6.5.2 语音处理陷阱

### 陷阱3：音频格式不兼容

**问题描述**：
```
问题：上传的音频格式不被Whisper模型支持
症状：
- API返回`Invalid Audio Format`错误
- pydub无法加载音频
- 语音识别失败

原因：
- 音频格式非WAV/MP3（如AAC、FLAC）
- 采样率不匹配（如44.1kHz vs 16kHz）
- 音频文件损坏
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **格式转换** | 使用ffmpeg转换为WAV/MP3 | 通用场景 |
| **重采样** | 使用pydub重采样为16kHz | Whisper模型 |
| **格式验证** | 上传前验证音频格式与采样率 | 前端/后端 |

**示例代码**：
```java
// 使用ffmpeg进行格式转换
public File convertToWAV(String inputPath) throws IOException {
    String outputPath = inputPath.replaceFirst("\\.[^.]+$", ".wav");
    
    ProcessBuilder builder = new ProcessBuilder(
        "ffmpeg", "-i", inputPath, "-ar", "16000", "-ac", "1", outputPath
    );
    builder.redirectErrorStream(true);
    
    Process process = builder.start();
    int exitCode = process.waitFor();
    
    if (exitCode != 0) {
        throw new IOException("FFmpeg conversion failed with exit code: " + exitCode);
    }
    
    return new File(outputPath);
}

// 采样率验证
public void validateSampleRate(File audioFile) {
    AudioInputStream audioStream = AudioSystem.getAudioInputStream(audioFile);
    AudioFormat format = audioStream.getFormat();
    
    // Whisper模型要求16kHz
    if (format.getSampleRate() != 16000) {
        throw new IllegalArgumentException("Audio sample rate must be 16000Hz, but got " + 
            format.getSampleRate() + "Hz");
    }
}
```

### 陷阱4：语音识别准确率低

**问题描述**：
```
问题：Whisper模型识别准确率低
症状：
- 识别结果错漏多
- 专业术语识别错误
- 方言识别失败

原因：
- 音频质量差（噪音大、音量低）
- 模型未微调（领域适应性差）
- 未使用 vad（语音活动检测）
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **音频预处理** | 降噪、音量归一化 | 音质差场景 |
| **领域微调** | 使用领域语料微调模型 | 专业领域场景 |
| **VAD过滤** | 使用vad过滤静音段 | 静音段多场景 |

**示例代码**：
```java
// 音频降噪（简化版）
public File denoiseAudio(File inputAudio) throws IOException {
    String outputPath = inputAudio.getPath().replaceFirst("\\.[^.]+$", "_denoised.wav");
    
    ProcessBuilder builder = new ProcessBuilder(
        "ffmpeg", "-i", inputAudio.getPath(), 
        "-af", "afftdn=nf=-20", // 降噪滤镜
        outputPath
    );
    builder.redirectErrorStream(true);
    
    Process process = builder.start();
    process.waitFor();
    
    return new File(outputPath);
}

// 音量归一化
public File normalizeVolume(File inputAudio) throws IOException {
    String outputPath = inputAudio.getPath().replaceFirst("\\.[^.]+$", "_normalized.wav");
    
    ProcessBuilder builder = new ProcessBuilder(
        "ffmpeg", "-i", inputAudio.getPath(), 
        "-af", "loudnorm=I=-16:TP=-1.5:LRA=11", // 立体声归一化
        outputPath
    );
    builder.redirectErrorStream(true);
    
    Process process = builder.start();
    process.waitFor();
    
    return new File(outputPath);
}
```

## 6.5.3 多模态Agent陷阱

### 陷阱5：模态融合失败

**问题描述**：
```
问题：多模态输入（图像+文本）融合失败
症状：
- LLM无法理解多模态输入
- 模态权重分配不合理
- 信息丢失

原因：
- 模态特征未对齐
- 融合策略不当（拼接 vs 加权）
- 提示词设计不合理
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **特征对齐** | 统一模态向量维度（如512维） | 通用场景 |
| **加权融合** | 根据模态重要性分配权重 | 多模态任务 |
| **提示词优化** | 设计多模态提示词模板 | 人类理解场景 |

**示例代码**：
```java
// 多模态特征对齐
public Vector alignModalities(Vector imageEmbedding, Vector textEmbedding) {
    // 统一维度（如512维）
    int targetDim = 512;
    
    if (imageEmbedding.dim() != targetDim) {
        imageEmbedding = resizeVector(imageEmbedding, targetDim);
    }
    
    if (textEmbedding.dim() != targetDim) {
        textEmbedding = resizeVector(textEmbedding, targetDim);
    }
    
    return imageEmbedding;
}

// 加权融合
public Vector fuseModalities(Vector imageEmbedding, Vector textEmbedding, 
                            double imageWeight, double textWeight) {
    // 检查维度
    if (imageEmbedding.dim() != textEmbedding.dim()) {
        throw new IllegalArgumentException("Embedding dimensions must match");
    }
    
    // 加权融合
    Vector fused = new Vector(imageEmbedding.dim());
    for (int i = 0; i < imageEmbedding.dim(); i++) {
        fused.set(i, imageEmbedding.get(i) * imageWeight + 
                     textEmbedding.get(i) * textWeight);
    }
    
    return fused;
}

// 多模态提示词模板
public String buildMultimodalPrompt(String textQuery, List<String> imageCaptions) {
    StringBuilder prompt = new StringBuilder();
    
    prompt.append("【多模态问题】\n");
    prompt.append("用户问题：").append(textQuery).append("\n\n");
    
    prompt.append("【相关图像】\n");
    for (int i = 0; i < imageCaptions.size(); i++) {
        prompt.append("图像").append(i + 1).append("：").append(imageCaptions.get(i)).append("\n");
    }
    
    prompt.append("\n【任务】\n");
    prompt.append("请根据以上图像描述回答用户问题。\n");
    prompt.append("注意：\n");
    prompt.append("1. 优先参考图像描述中的信息\n");
    prompt.append("2. 结合用户问题的意图\n");
    prompt.append("3. 如果图像描述不足，请说明\n");
    
    return prompt.toString();
}
```

### 陷阱6：模态延迟不一致

**问题描述**：
```
问题：不同模态处理耗时差异大（图像慢、文本快）
症状：
- 响应延迟不一致
- 用户体验差
- 请求堆积

原因：
- 图像处理耗时长（CLIP向量化）
- 语音识别耗时长（Whisper转录）
- 未并行处理模态
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **并行处理** | 同时处理图像、文本、语音 | 高并发场景 |
| **预加载模型** | 启动时加载模型到GPU | GPU可用场景 |
| **异步响应** | 先返回文本结果，图像结果后补 | 用户体验场景 |

**示例代码**：
```java
// 并行处理
public MultimodalResponse processMultimodalAsync(MultimodalRequest request) {
    // 1. 并行处理图像、文本、语音
    CompletableFuture<Embedding> imageFuture = CompletableFuture.supplyAsync(
        () -> imageEncoder.encode(request.getImage())
    );
    
    CompletableFuture<Vector> textFuture = CompletableFuture.supplyAsync(
        () -> textEncoder.encode(request.getText())
    );
    
    CompletableFuture<String> speechFuture = CompletableFuture.supplyAsync(
        () -> speechRecognizer.transcribe(request.getAudio())
    );
    
    // 2. 等待所有模态处理完成
    CompletableFuture.allOf(imageFuture, textFuture, speechFuture).join();
    
    // 3. 融合多模态结果
    Embedding imageEmbedding = imageFuture.join();
    Vector textVector = textFuture.join();
    String speechTranscript = speechFuture.join();
    
    // 4. 调用LLM生成响应
    return llm.generate(buildMultimodalPrompt(textVector, imageEmbedding, speechTranscript));
}
```

## 6.5.4 通用调试技巧

### 陷阱7：多模态调试信息不足

**问题描述**：
```
问题：多模态调试缺乏足够信息定位问题
症状：
- 难以复现问题
- 日志信息不足
- 无法追踪模态处理流程
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **增强日志** | 记录模态处理各阶段耗时 | 开发/测试环境 |
| **可视化调试** | 输出中间结果（如图像、音频） | 开发/测试环境 |
| **性能监控** | 记录各模态处理耗时 | 生产环境 |

**示例代码**：
```java
// 增强日志
logger.info("Image encoding started - image size: {}x{}", width, height);
long start = System.currentTimeMillis();
Embedding embedding = clipModel.encode(image);
long duration = System.currentTimeMillis() - start;
logger.info("Image encoding completed - duration: {}ms, embedding dim: {}", 
    duration, embedding.dim());

// 可视化调试
public void visualizeDebug(String inputPath, BufferedImage processedImage) {
    try {
        ImageIO.write(processedImage, "jpg", new File(inputPath + "_debug.jpg"));
        logger.info("Debug image saved: {}", inputPath + "_debug.jpg");
    } catch (IOException e) {
        logger.warn("Failed to save debug image", e);
    }
}

// 性能监控
public class MultimodalMetrics {
    private static final ThreadLocal<Map<String, Long>> metrics = 
        ThreadLocal.withInitial(HashMap::new);
    
    public static void record(String key, long duration) {
        metrics.get().put(key, duration);
    }
    
    public static Map<String, Long> getMetrics() {
        return new HashMap<>(metrics.get());
    }
}
```
