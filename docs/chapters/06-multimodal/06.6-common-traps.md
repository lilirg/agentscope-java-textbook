## 6.5.1 图片输入相关陷阱

### 6.5.1.1 陷阱1：OpenCV库加载失败

**现象**：
```
Error: java.lang.UnsatisfiedLinkError: no opencv_java490 in java.library.path
```

**原因**：
- OpenCV native library未正确加载
- `System.loadLibrary()`找不到动态链接库（DLL/SO/DYLIB）

**调试方法**：
```java
// 1. 检查OpenCV是否加载成功
try {
    System.loadLibrary(Core.NATIVE_LIBRARY_NAME);
    System.out.println("OpenCV加载成功");
} catch (UnsatisfiedLinkError e) {
    System.err.println("OpenCV加载失败：" + e.getMessage());
    
    // 2. 检查java.library.path
    System.out.println("java.library.path: " + System.getProperty("java.library.path"));
    
    // 3. 尝试直接加载绝对路径
    // System.load("C:\\opencv\\build\\java\\x64\\opencv_java490.dll");
}
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **Maven依赖** | 使用`org.openpnp:opencv`自动加载 | 开发/测试 |
| **手动加载** | 指定native library路径 | 生产环境 |
| **Docker** | 使用OpenCV预构建镜像 | 容器化部署 |

**示例**：
```java
public class OpenCVLoader {
    static {
        String os = System.getProperty("os.name").toLowerCase();
        String libPath;
        
        if (os.contains("win")) {
            libPath = "C:\\opencv\\build\\java\\x64\\opencv_java490.dll";
        } else if (os.contains("mac")) {
            libPath = "/usr/local/lib/libopencv_java490.dylib";
        } else if (os.contains("linux")) {
            libPath = "/usr/local/lib/libopencv_java490.so";
        } else {
            throw new RuntimeException("不支持的操作系统");
        }
        
        try {
            System.load(libPath);
            System.out.println("OpenCV加载成功");
        } catch (UnsatisfiedLinkError e) {
            throw new RuntimeException("OpenCV native library加载失败: " + libPath, e);
        }
    }
}
```

### 6.5.1.2 陷阱2：图片格式不支持

**现象**：
```
Error: OpenCV(4.9.0) error: (-5:Bad argument) in function 'imdecode'
> Invalid input data
```

**原因**：
- 图片数据损坏（字节数组无效）
- 图片格式不支持（如：HEIC、WebP）

**调试方法**：
```java
// 1. 检查图片数据
public static void validateImage(byte[] imageData) {
    if (imageData == null || imageData.length == 0) {
        throw new IllegalArgumentException("图片数据为空");
    }
    
    // 2. 检查文件头（Magic Number）
    if (imageData.length < 4) {
        throw new IllegalArgumentException("图片数据过短");
    }
    
    // JPEG: FF D8 FF
    if (imageData[0] == (byte)0xFF && imageData[1] == (byte)0xD8 && imageData[2] == (byte)0xFF) {
        System.out.println("图片格式：JPEG");
    }
    // PNG: 89 50 4E 47
    else if (imageData[0] == (byte)0x89 && imageData[1] == (byte)0x50 && imageData[2] == (byte)0x4E && imageData[3] == (byte)0x47) {
        System.out.println("图片格式：PNG");
    }
    // GIF: 47 49 46 38
    else if (imageData[0] == (byte)0x47 && imageData[1] == (byte)0x49 && imageData[2] == (byte)0x46 && imageData[3] == (byte)0x38) {
        System.out.println("图片格式：GIF");
    }
    else {
        System.out.println("图片格式：未知");
    }
}

// 3. 使用OpenCV imdecode检查
public static boolean canDecode(byte[] imageData) {
    Mat mat = Imgcodecs.imdecode(new MatOfByte(imageData), Imgcodecs.IMREAD_COLOR);
    boolean valid = !mat.empty();
    mat.release();
    return valid;
}
```

**解决方案**：
```java
public class ImagePreprocessor {
    public static Mat decodeImage(byte[] imageData) {
        validateImage(imageData);
        
        Mat mat = Imgcodecs.imdecode(new MatOfByte(imageData), Imgcodecs.IMREAD_COLOR);
        if (mat.empty()) {
            throw new IllegalArgumentException("图片解码失败：数据无效或格式不支持");
        }
        return mat;
    }
}
```

## 6.5.2 语音输入相关陷阱

### 6.5.2.1 陷阱3：pydub转换音频失败

**现象**：
```
Error: pydub.exceptions.CouldntDecodeError: Couldn't decode file
```

**原因**：
- 音频格式不支持（如：WMA、AAC）
- ffmpeg未安装（pydub依赖ffmpeg）

**调试方法**：
```python
## 1. 检查ffmpeg安装
import subprocess
try:
    result = subprocess.run(['ffmpeg', '-version'], capture_output=True, text=True)
    print("ffmpeg版本：", result.stdout.split('\n')[0])
except FileNotFoundError:
    print("ffmpeg未安装")
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **安装ffmpeg** | 安装ffmpeg并加入PATH | 开发/测试 |
| **使用mp3格式** | 限制上传格式为mp3 | 简单场景 |
| **Java音频处理** | 使用Java AudioSystem代替pydub | 无Python环境 |

**示例**：
```bash
## 安装ffmpeg（Windows）
## 下载：https://www.gyan.dev/ffmpeg/builds/
## 添加到PATH

## 安装ffmpeg（Linux）
sudo apt-get install ffmpeg

## 安装ffmpeg（Mac）
brew install ffmpeg
```

**Java音频处理替代方案**：
```java
import javax.sound.sampled.*;

public class AudioPreprocessor {
    public static byte[] convertToWav(byte[] audioData) {
        try {
            AudioInputStream inputStream = AudioSystem.getAudioInputStream(new ByteArrayInputStream(audioData));
            AudioFormat baseFormat = inputStream.getFormat();
            
            // 转换为WAV格式
            AudioFormat decodedFormat = new AudioFormat(
                AudioFormat.Encoding.PCM_SIGNED,
                baseFormat.getSampleRate(),
                16, // 16-bit
                baseFormat.getChannels(),
                baseFormat.getChannels() * 2,
                baseFormat.getSampleRate(),
                false
            );
            
            AudioInputStream decodedInputStream = AudioSystem.getAudioInputStream(decodedFormat, inputStream);
            ByteArrayOutputStream outputStream = new ByteArrayOutputStream();
            AudioSystem.write(decodedInputStream, AudioFileFormat.Type.WAVE, outputStream);
            
            return outputStream.toByteArray();
        } catch (Exception e) {
            throw new RuntimeException("音频格式转换失败", e);
        }
    }
}
```

```java
// 2. 监控处理时间
long startTime = System.currentTimeMillis();
// Whisper API调用
long endTime = System.currentTimeMillis();
System.out.println("处理耗时：" + (endTime - startTime) + "ms");
```

**示例**：

## 6.5.3 多模态系统通用陷阱

### 6.5.3.1 陷阱5：内存溢出（OOM）

**现象**：
```
Error: java.lang.OutOfMemoryError: Java heap space
```

**原因**：
- 图片未释放（Mat未调用`release()`）
- 音频分段未清理（临时文件过多）

**调试方法**：
```java
// 1. 监控内存使用
Runtime runtime = Runtime.getRuntime();
System.out.println("Max Memory: " + runtime.maxMemory() / (1024 * 1024) + "MB");
System.out.println("Used Memory: " + (runtime.totalMemory() - runtime.freeMemory()) / (1024 * 1024) + "MB");

// 2. 检查OpenCV Mat释放
Mat mat = new Mat();
// ... 处理
mat.release(); // 必须释放
System.out.println("Mat是否已释放：" + mat.empty());
```

**解决方案**：
```java
public class ImagePreprocessor {
    public static float[] preprocess(byte[] imageData) {
        Mat mat = null;
        try {
            mat = decodeImage(imageData);
            Mat resized = resizeImage(mat, 224, 224);
            return normalizeImage(resized);
        } finally {
            // 确保释放
            if (mat != null) mat.release();
        }
    }
}
```

### 6.5.3.2 陷阱6：线程安全问题

**现象**：
```
Error: OpenCV(4.9.0) error: (-215:Assertion failed) !_src.empty() in function 'cv::cvtColor'
```

**原因**：
- 多线程共享OpenCV Mat对象
- 图片/音频预处理并发执行

**调试方法**：
```java
// 1. 使用ThreadLocal隔离
public class ImagePreprocessor {
    private static final ThreadLocal<Mat> threadMat = ThreadLocal.withInitial(Mat::new);
    
    public static Mat decodeImage(byte[] imageData) {
        Mat mat = threadMat.get();
        // ... 处理
        return mat;
    }
}

// 2. 使用synchronized同步
public class SpeechPreprocessor {
    private static final Object lock = new Object();
    
    public static String[] segmentAudio(String audioPath) {
        synchronized (lock) {
            // ... 处理
        }
    }
}
```

**解决方案**：
```java
public class MultimodalInput {
    public static MultimodalInput ofImage(byte[] image) {
        // 深拷贝，避免引用共享
        return new MultimodalInput(null, Arrays.copyOf(image, image.length), null);
    }
    
    public static MultimodalInput ofAudio(byte[] audio) {
        // 深拷贝，避免引用共享
        return new MultimodalInput(null, null, Arrays.copyOf(audio, audio.length));
    }
}
```
