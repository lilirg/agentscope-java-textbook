# 1.3 核心概念

> **本节学习目标**：掌握Agent开发的核心原理

---

## 1.3.1 Agent核心概念

### 1.3.1.1 Agent定义

**Agent（代理）** 是能够感知环境、推理决策、执行行动的智能实体。

```
Agent = 感知 + 推理 + 决策 + 行动
```

**Agent核心特性**：
| 特性 | 说明 |
|------|------|
| **感知能力** | 接收外部输入（文本、语音、图像） |
| **推理能力** | 基于知识库和模型进行推理 |
| **决策能力** | 选择最优行动方案 |
| **行动能力** | 执行具体操作（API调用、文件操作） |

### 1.3.1.2 Agent与LLM的关系

**LLM** 是Agent的核心组件，但不是全部：

```
Agent = LLM + Memory + Tool + Planning + Execution
```

**LLM能力**：
- ✅ 理解自然语言
- ✅ 生成文本
- ✅ 推理
- ❌ 无法执行外部操作
- ❌ 无法访问外部知识

**Agent能力**：
- ✅ LLM推理
- ✅ API调用
- ✅ 外部知识访问
- ✅ 具体操作执行

---

## 1.3.2 提示词工程

### 1.3.2.1 提示词原理

**提示词（Prompt）** 是引导LLM生成期望输出的指令。

```
Prompt = 指令 + 上下文 + 示例
```

**提示词组成部分**：
| 部分 | 说明 |
|------|------|
| **指令** | 明确的任务目标 |
| **上下文** | 相关背景信息 |
| **示例** | 示例输入输出对 |

**提示词工作流程**：
```
用户输入 → 提示词工程 → LLM推理 → 输出
```

---

## 1.3.3 API接入

### 1.3.3.1 API基础

**API（Application Programming Interface）** 是应用程序之间通信的接口。

```
API = 接口定义 + 请求/响应
```

**API核心概念**：
| 概念 | 说明 |
|------|------|
| **接口定义** | 定义应用程序间的交互方式 |
| **HTTP协议** | HTTP/HTTPS传输协议 |
| **请求/响应** | 请求（Request）与响应（Response）格式 |

### 1.3.3.2 RESTful API

**RESTful API** 是基于REST架构的API设计风格。

```
RESTful API = 资源 + HTTP方法 + 状态码
```

**RESTful API核心概念**：
| 概念 | 说明 |
|------|------|
| **资源** | 通过URL标识的实体 |
| **HTTP方法** | GET（查询）、POST（创建）、PUT（更新）、DELETE（删除） |
| **状态码** | 200（成功）、400（错误）、500（服务器错误） |

## 1.3.4 Agent架构

### 1.3.4.1 核心架构

**Agent架构** 是AI Agent的核心结构，定义Agent的组织方式与运行机制。

```
Agent架构 = 核心组件 + 运行机制 + 协作模式
```

**Agent核心组件**：
| 组件 | 说明 |
|------|------|
| **Memory** | 记忆系统，存储历史信息 |
| **Tool** | 工具系统，调用外部API |
| **Planning** | 计划系统，制定行动方案 |
| **Execution** | 执行系统，执行具体操作 |

**Agent运行机制**：
```
感知环境 → 推理决策 → 执行行动 → 反馈学习
```

**Agent协作模式**：
| 模式 | 说明 |
|------|------|
| **单Agent** | 单个Agent独立工作 |
| **多Agent** | 多个Agent协作工作 |
