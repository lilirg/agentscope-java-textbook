## AGENTS.md：Agent定义与协作规范

### AGENTS.md核心内容

```markdown
# Agent协作规范

## 1. Agent定义

### SalesAgent
- **角色**：销售顾问
- **职责**：接收订单、检查库存、生成订单号
- **输入**：用户订单请求
- **输出**：订单号或错误信息

### ServiceAgent
- **角色**：客服助手
- **职责**：回答用户咨询、检索知识库
- **输入**：用户咨询
- **输出**：回答或"暂无相关信息"

---

## 2. 协作流程

```
用户输入
    ↓
[ManagerAgent] 意图识别
    ↓
    ├───订单请求 → SalesAgent
    └───咨询请求 → ServiceAgent
    ↓
结果汇总 → 用户
```

## 实战：编写项目完整文档体系

### 任务

为第8章的「销售+客服双Agent系统」编写完整文档：

1. `ARCHITECTURE.md`  
2. `AGENTS.md`  
3. `README.md`  

### 示例：ARCHITECTURE.md

```markdown
# 销售+客服双Agent系统架构文档

## 1. 项目简介

### 项目是什么？
一个基于多Agent协作的订单处理系统，能够自动识别用户意图（下单/咨询）并分发给对应Agent。

### 解决什么问题？
- 传统客服系统只能处理单一任务
- 无法区分订单与咨询请求
- 需要人工干预分发

### 目标用户
- 中小型电商平台
- 需要自动化客服的SaaS服务商

---

## 2. 系统架构

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    User Interface                            │
│  [下单按钮] [咨询按钮]                                       │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                    Manager Agent                             │
│  - 意图识别（LLM）                                           │
│  - 分发给SalesAgent或ServiceAgent                            │
│  - 汇总结果                                                   │
└─────────────────────────────────────────────────────────────┘
                    ↓ 分发
┌─────────────────────────────────────────────────────────────┐
│              SalesAgent / ServiceAgent                       │
│  ┌──────────────┐      ┌──────────────┐                    │
│  │ SalesAgent   │      │ ServiceAgent │                    │
│  │ - 接收订单   │      │ - 接收咨询   │                    │
│  │ - 库存检查   │      │ - 知识库检索 │                    │
│  │ - 生成订单号 │      │ - 生成回答   │                    │
│  └──────────────┘      └──────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 职责 | 实现 |
|------|------|------|
| ManagerAgent | 意图识别、任务分发 | LLM分类 + 权重路由 |
| SalesAgent | 订单处理、库存检查 | OrderParser + InventoryAPI |
| ServiceAgent | 咨询解答、知识检索 | FAQ + RAG |

---

## 3. 数据流

### 主要流程

```
用户输入（下单/咨询）
    ↓
[ManagerAgent] 意图识别
    ↓
    ├───订单请求 → [SalesAgent] 解析订单 → 库存检查 → 生成订单号 → 返回结果
    └───咨询请求 → [ServiceAgent] RAG检索 → 生成回答 → 返回结果
    ↓
结果汇总
    ↓
返回用户
```

## 6. 架构决策

### 关键决策

| 决策 | 原因 | 权衡 |
|------|------|------|
| 采用Manager-Worker模式 | 任务类型明确、易于扩展 | 需要额外的意图识别逻辑 |
| 使用LLM做意图识别 | 灵活性高、无需训练 | 依赖LLM可用性 |
| Redis缓存会话 | 性能高、支持分布式 | 需额外维护Redis服务 |

---

## 7. 扩展指南

### 如何添加新Agent

1. 创建Agent类（实现`Agent`接口）  
2. 在`ManagerAgent`中添加路由规则  
3. 更新`ARCHITECTURE.md`与`AGENTS.md`

### 常见修改场景

| 场景 | 修改位置 |
|------|---------|
| 新增意图类型 | `ManagerAgent.detectIntent()` |
| 新增工具调用 | `Agent`类的`tools`字段 |
| 调整提示词 | `Agent.systemPrompt` |
