## 8.5.1 消息队列相关陷阱

### 8.5.1.1 陷阱1：Kafka连接失败

**现象**：
```
Error: org.apache.kafka.common.errors.TimeoutException: Topic tasks not available
```

**原因**：
- Kafka服务未启动
- 配置的bootstrap.servers地址错误
- 网络问题（防火墙、端口未开放）

**调试方法**：
```bash
# 1. 检查Kafka服务状态
docker ps | grep kafka

# 2. 检查Kafka topic
kafka-topics.sh --list --bootstrap-server localhost:9092

# 3. 检查网络连接
telnet localhost 9092
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **Docker启动** | 推荐方式，环境隔离 | 开发/测试 |
| **本地安装** | 高性能，无Docker依赖 | 生产环境 |
| **Mock测试** | 单元测试时模拟Kafka | CI/CD |

**示例**：
```java
public class MessageQueueClient {
    public MessageQueueClient() {
        String bootstrapServers = ConfigLoader.get("KAFKA_BOOTSTRAP_SERVERS", "localhost:9092");
        
        // 1. 健康检查
        if (!isKafkaReady(bootstrapServers)) {
            System.err.println("Kafka服务不可用，使用Mock模式");
            this.producer = new MockKafkaProducer(); // Mock方案
            this.consumer = new MockKafkaConsumer();
            return;
        }

        // 2. 正常初始化
        this.producer = new KafkaProducer<>(producerProps);
    }

    private boolean isKafkaReady(String bootstrapServers) {
        try {
            Properties props = new Properties();
            props.put("bootstrap.servers", bootstrapServers);
            KafkaProducer<String, String> testProducer = new KafkaProducer<>(props);
            testProducer.close();
            return true;
        } catch (Exception e) {
            return false;
        }
    }
}
```

### 8.5.1.2 陷阱2：消息丢失

**现象**：
```
Warning: Message msg_123456 lost (no consumer)
```

**原因**：
- Consumer未订阅Topic
- 消费者组配置错误
- 消息过期（TTL）

**调试方法**：
```bash
# 1. 检查Consumer订阅情况
kafka-consumers.sh --bootstrap-server localhost:9092 --group my-group --describe

# 2. 检查Topic消息积压
kafka-topics.sh --bootstrap-server localhost:9092 --describe --topic tasks

# 3. 检查Consumer日志
tail -f /var/log/kafka/consumer.log
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **检查订阅** | 确保Consumer订阅正确的Topic | 通用 |
| **设置TTL** | 避免消息过期 | 高频消息场景 |
| **消息持久化** | 启用消息持久化 | 关键消息 |

**示例**：
```java

            }
```

## 8.5.2 Agent协作相关陷阱

### 8.5.2.1 陷阱3：调度策略不生效

**现象**：
```
Warning: RoundRobinStrategy not working (all tasks to first worker)
```

**原因**：
- `SchedulingStrategy`未正确注入
- `selectWorker()`方法未被调用

**调试方法**：
```java
// 1. 检查依赖注入
@Autowired
private SchedulingStrategy schedulingStrategy;

@PostConstruct
public void logStrategy() {
    System.out.println("调度策略：" + schedulingStrategy.getClass().getName());
}

// 2. 检查方法调用
public void handleRequest(String request) {
    String worker = schedulingStrategy.selectWorker(workers, null);
    System.out.println("选中Worker：" + worker);
}
```

**解决方案**：
```java
// 正确的依赖注入方式
@Component
public class ManagerAgent {
    private final SchedulingStrategy schedulingStrategy;

    public ManagerAgent(SchedulingStrategy schedulingStrategy) {
        this.schedulingStrategy = schedulingStrategy;
    }

    public void handleRequest(String request) {
        String worker = schedulingStrategy.selectWorker(workers, null);
        // ...
    }
}
```

### 8.5.2.2 陷阱4：Agent间消息循环依赖

**现象**：
```
Error: StackOverflowError (infinite recursion)
```

**原因**：
- Manager→Worker→Manager→Worker...形成循环
- 消息响应未正确终止

**调试方法**：
```java
// 1. 添加消息追踪
public class Message {
    private List<String> trace = new ArrayList<>();

    public void addTrace(String agent) {
        trace.add(agent);
        if (trace.size() > 10) {
            throw new IllegalStateException("消息循环依赖 detected: " + trace);
        }
    }
}

// 2. 设置消息最大转发次数
public void processMessage(Message message) {
    if (message.getForwardCount() > 5) {
        throw new IllegalArgumentException("消息转发次数过多");
    }
    // ...
}
```

**解决方案**：
```java
public class Message {
    private int forwardCount = 0;

    public void incrementForwardCount() {
        if (++forwardCount > 5) {
            throw new IllegalStateException("消息转发次数超过限制");
        }
    }
}
```

## 8.5.3 通用调试技巧

### 8.5.3.1 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class ManagerAgent {
    private static final Logger logger = LoggerFactory.getLogger(ManagerAgent.class);

    public void handleRequest(String request) {
        logger.info("Processing request: {}", request);
        
        try {
            // ... 业务逻辑
        } catch (Exception e) {
            logger.error("Failed to process request: {}", request, e);
            throw e;
        }
    }
}
```

### 8.5.3.2 单元测试（JUnit + Mockito）

**测试示例**：
```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;
import static org.junit.jupiter.api.Assertions.*;

public class RoundRobinStrategyTest {
    @Test
    public void testSelectWorker() {
        // 1. 创建Mock
        RoundRobinStrategy strategy = new RoundRobinStrategy();
        List<String> workers = Arrays.asList("W1", "W2", "W3");

        // 2. 执行
        String worker1 = strategy.selectWorker(workers, null);
        String worker2 = strategy.selectWorker(workers, null);
        String worker3 = strategy.selectWorker(workers, null);

        // 3. 验证
        assertEquals("W1", worker1);
        assertEquals("W2", worker2);
        assertEquals("W3", worker3);
    }
}
```
