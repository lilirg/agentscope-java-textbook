### 8.6.2 代码改错（每题15分，共30分）

#### 1. 调度策略改错

```java
// ❌ 错误代码（RoundRobinStrategy.java）
public class RoundRobinStrategy implements SchedulingStrategy {
    private static int counter = 0; // ❌ 静态变量，多线程不安全

    @Override
    public String selectWorker(List<String> workers, Message message) {
        int index = counter % workers.size(); // ❌ 未使用原子操作
        counter++; // ❌ 非线程安全
        return workers.get(index);
    }
}
```

**问题**：  
1. 静态变量导致多线程竞争  
2. `counter++`非原子操作

**正确代码**：
```java
public class RoundRobinStrategy implements SchedulingStrategy {
    private final AtomicInteger counter = new AtomicInteger(0);

    @Override
    public String selectWorker(List<String> workers, Message message) {
        int index = counter.getAndIncrement() % workers.size(); // ✅ 使用原子操作
        return workers.get(index);
    }
}
```

#### 2. Manager Agent改错

```java
// ❌ 错误代码（ManagerAgent.java）
public class ManagerAgent {
    private SchedulingStrategy schedulingStrategy; // ❌ 未注入

    public void handleRequest(String request) {
        String worker = schedulingStrategy.selectWorker(workers, null); // ❌ NPE
        // ...
    }
}
```

**问题**：  
1. `schedulingStrategy`未注入，导致NPE  
2. 缺少依赖注入注解

**正确代码**：
```java
@Component
public class ManagerAgent {
    private final SchedulingStrategy schedulingStrategy;

    public ManagerAgent(@Autowired SchedulingStrategy schedulingStrategy) {
        this.schedulingStrategy = schedulingStrategy;
    }

    public void handleRequest(String request) {
        String worker = schedulingStrategy.selectWorker(workers, null); // ✅ 正确注入
        // ...
    }
}
```

### 8.6.3 小型设计（30分）

#### 设计任务：实现RabbitMQ消息队列客户端

**需求**：  
设计一个`RabbitMQClient`类，实现消息发送与接收。

**功能要求**：  
1. 支持交换机模式（Direct/Fanout/Topic）  
2. 支持消息持久化  
3. 支持ACK确认机制  

**参考答案**：
```java
package com.textbook.chapter8.client;

import com.rabbitmq.client.Channel;
import com.rabbitmq.client.Connection;
import com.rabbitmq.client.ConnectionFactory;

import java.io.IOException;
import java.util.concurrent.TimeoutException;

/**
 * RabbitMQ消息队列客户端
 * @since v5.18.0
 */
public class RabbitMQClient {
    private final Connection connection;
    private final Channel channel;

    public RabbitMQClient() throws IOException, TimeoutException {
        // 1. 创建连接工厂
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost(ConfigLoader.get("RABBITMQ_HOST", "localhost"));
        factory.setPort(Integer.parseInt(ConfigLoader.get("RABBITMQ_PORT", "5672")));
        
        // 2. 创建连接
        this.connection = factory.newConnection();
        this.channel = connection.createChannel();
    }

    /**
     * 发送消息
     * @param exchange 交换机名称
     * @param routingKey 路由键
     * @param message 消息内容
     */
    public void publish(String exchange, String routingKey, String message) throws IOException {
        // 消息持久化
        channel.basicPublish(exchange, routingKey, 
            null, // props
            message.getBytes());
    }

    /**
     * 订阅消息
     * @param queue 队列名称
     * @param callback 回调接口
     */
    public void subscribe(String queue, Consumer<String> callback) throws IOException {
        channel.basicConsume(queue, true, (consumerTag, delivery) -> {
            String message = new String(delivery.getBody());
            callback.accept(message);
        }, consumerTag -> {});
    }

    /**
     * 关闭连接
     */
    public void close() throws IOException, TimeoutException {
        channel.close();
        connection.close();
    }
}
```

### 8.6.4 拓展挑战（20分）

#### 挑战1：调度策略优化

**背景**：  
当前轮询调度无法考虑Worker负载，导致低性能Worker负载过高。

**问题**：  
1. 如何实现基于负载的调度算法？  
2. 如何收集Worker的实时负载信息？  
3. 如何实现动态调度（根据负载调整分配）？

**参考答案**：

```java
public class LoadBalancingStrategy implements SchedulingStrategy {
    private final Map<String, Integer> workerLoad = new ConcurrentHashMap<>();

    @Override
    public String selectWorker(List<String> workers, Message message) {
        // 1. 获取负载最低的Worker
        return workers.stream()
            .min(Comparator.comparingInt(workerLoad::get))
            .orElse(workers.get(0));
    }

    /**
     * 更新Worker负载
     * @param worker Worker名称
     * @param load 负载值（任务数量）
     */
    public void updateLoad(String worker, int load) {
        workerLoad.put(worker, load);
    }
}
```

**负载收集方式**：
- Worker定期上报负载（心跳机制）
- Manager定期查询Worker负载
- 基于消息队列积压量估算负载

#### 挑战2：多Agent系统性能优化

**背景**：  
当前多Agent系统在高并发下性能下降。

**问题**：  
1. 如何优化消息队列性能？  
2. 如何减少Agent间通信开销？  
3. 如何实现Agent池化（避免频繁创建/销毁）？

**参考答案**：

```java
public class AgentPool {
    private final ConcurrentLinkedQueue<WorkerAgent> availableAgents = new ConcurrentLinkedQueue<>();
    private final int poolSize;

    public AgentPool(int poolSize) {
        this.poolSize = poolSize;
        for (int i = 0; i < poolSize; i++) {
            availableAgents.offer(new WorkerAgent("Worker-" + i));
        }
    }

    public WorkerAgent borrow() {
        return availableAgents.poll();
    }

    public void returnAgent(WorkerAgent agent) {
        availableAgents.offer(agent);
    }
}
```

**优化策略**：
- 消息队列：使用异步发送、批处理
- Agent池化：预创建Agent池，避免频繁创建
- 本地通信：同一JVM内使用内存队列
