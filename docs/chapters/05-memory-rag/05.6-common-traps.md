### 5.5.1 短期记忆相关陷阱

#### 5.5.1.1 陷阱1：上下文窗口溢出（Context Window Overflow）

**现象**：
```
Error: This model's maximum context length is 128000 tokens, 
but your prompt contains 135478 tokens.
```

**原因**：
- 短期记忆窗口大小（20条） × 每条消息平均长度（200 tokens） = 4000 tokens  
- 加上RAG检索结果（3 × 500 tokens = 1500 tokens）  
- 再加上Prompt模板与用户输入  
- **总计可能超出128K限制**

**调试方法**：
```java
// 1. 统计token数量
public int countTokens(String text) {
    // 使用tiktoken-java或tiktoken4j
    // TODO: 请对照官方v0.2.0文档验证
    return text.split("\\s+").length * 1.3; // 粗略估算（1词≈1.3 tokens）
}

// 2. 添加日志
shortTermMemory.getRecent().forEach(msg -> {
    System.out.println("Message: " + msg.getContent());
    System.out.println("Tokens: " + countTokens(msg.getContent()));
});
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **压缩摘要** | 用LLM生成对话摘要 | 对话轮次多（>20） |
| **截断正文** | 保留摘要，丢弃正文 | 对话内容冗长 |
| **降低窗口** | 将窗口从20降至10 | API成本敏感 |

**示例**：
```java
public class CompressedShortTermMemory extends ShortTermMemory {
    private final LLM llm; // 调用LLM生成摘要

    @Override
    public void add(Message msg) {
        super.add(msg);
        
        // 如果总token数超出阈值
        if (calculateTotalTokens() > 100000) {
            // 生成摘要
            List<Message> recent = getRecent();
            String summary = llm.generate("将以下对话压缩为3句话摘要：\n" + 
                recent.stream().map(Message::toString).collect(Collectors.joining("\n")));
            
            // 清空历史，替换为摘要
            clear();
            add(new Message("system", "[对话摘要]" + summary));
        }
    }
}
```

#### 5.5.1.2 陷阱2：对话顺序混乱（Message Order）

**现象**：
```
用户：你好
Agent：你好！有什么可以帮您？
用户：北京天气？
用户：后天呢？  ← 顺序错乱！
```

**原因**：
- 多线程并发调用 `add()` 方法（如：WebSocket + HTTP双通道输入）
- LinkedList不是线程安全的

**调试方法**：
```java
// 1. 添加日志
public void add(Message msg) {
    System.out.println("Adding message: " + msg + " at index " + history.size());
    history.addLast(msg);
}

// 2. 检查最终顺序
for (int i = 0; i < history.size(); i++) {
    System.out.println(i + ": " + history.get(i));
}
```

**解决方案**：
```java
public class ThreadSafeShortTermMemory {
    private final LinkedList<Message> history = new LinkedList<>();
    private final int maxMessages = 20;
    private final Object lock = new Object(); // 显式锁

    public void add(Message msg) {
        synchronized (lock) {
            history.addLast(msg);
            while (history.size() > maxMessages) {
                history.removeFirst();
            }
        }
    }

    public List<Message> getRecent() {
        synchronized (lock) {
            return new java.util.ArrayList<>(history);
        }
    }
}
```

#### 5.5.2.1 陷阱3：JSON序列化失败

**现象**：

```java
// 1. 打印JSON字符串
String json = memory.toJson();
```

```java
public class LongTermMemory {

    public LongTermMemory() {}

    // 2. 字段类型明确
    private String userId;
```

#### 5.5.2.2 陷阱4：Redis缓存穿透（Cache Miss）

**现象**：
```
Error: java.net.ConnectException: Connection refused (Connection refused)
```

**原因**：
- Redis未启动或连接失败
- Key不存在（首次访问用户）

**调试方法**：
```bash
# 检查Redis状态
redis-cli ping
# 预期输出：PONG

# 检查Key是否存在
redis-cli GET user:u123456
# 预期输出：nil（表示不存在）或JSON数据
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **降级到文件** | Redis失败时读取JSON文件 | 开发/测试环境 |
| **缓存空值** | Key不存在时存储空值（防止穿透） | 高并发生产环境 |
| **健康检查** | 启动时验证Redis连接 | 生产环境 |

**示例**：
```java
public class LongTermMemoryCache {
    private final RedisTemplate<String, String> redisTemplate;
    private final String baseDir = "data/long-term-memory/";

    public LongTermMemory get(String userId) {
        try {
            String json = redisTemplate.opsForValue().get("user:" + userId);
            if (json != null) {
                return LongTermMemory.fromJson(json);
            } else {
                // 缓存空值（防止穿透）
                redisTemplate.opsForValue().set("user:" + userId, "", 60, TimeUnit.SECONDS);
                return null;
            }
        } catch (Exception e) {
            System.err.println("Redis连接失败，降级到文件存储");
            return getFromFile(userId); // 降级方案
        }
    }

    private LongTermMemory getFromFile(String userId) {
        String filePath = baseDir + "user-" + userId + ".json";
        File file = new File(filePath);
        if (!file.exists()) {
            return null;
        }
        try {
            String json = Files.readString(Paths.get(filePath));
            return LongTermMemory.fromJson(json);
        } catch (Exception e) {
            throw new RuntimeException("文件读取失败", e);
        }
    }
}
```

### 5.5.3 RAG相关陷阱

#### 5.5.3.1 陷阱5：Chroma连接超时

**现象**：
```
Error: java.net.SocketTimeoutException: Connect timeout exceeded
```

**原因**：
- Chroma服务未启动
- 端口配置错误（默认8000）
- 防火墙拦截

**调试方法**：
```bash
# 检查Chroma服务状态
curl http://localhost:8000/api/v2/is_ready
# 预期输出：{"status":"ok"}

# 检查端口监听
netstat -ano | findstr :8000  # Windows
lsof -i :8000                # Linux/Mac
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **Docker启动** | 推荐方式，环境隔离 | 开发/测试 |
| **本地编译** | 高性能，无Docker依赖 | 生产环境 |
| **Mock测试** | 单元测试时模拟Chroma | CI/CD |

**示例**：
```java
public class RAGRepository {
    private final ChromaClient client;

    public RAGRepository() {
        String chromaEndpoint = ConfigLoader.get("CHROMA_ENDPOINT");
        
        // 1. 健康检查
        if (!isChromaReady(chromaEndpoint)) {
            System.err.println("Chroma服务不可用，使用Mock模式");
            this.client = new MockChromaClient(); // Mock方案
            return;
        }

        // 2. 正常初始化
        this.client = new ChromaClient(chromaEndpoint);
    }

    private boolean isChromaReady(String endpoint) {
        try {
            HttpURLConnection conn = (HttpURLConnection) new URL(endpoint + "/api/v2/is_ready").openConnection();
            conn.setRequestMethod("GET");
            return conn.getResponseCode() == 200;
        } catch (Exception e) {
            return false;
        }
    }
}
```

#### 5.5.3.2 陷阱6：向量化模型内存溢出

**现象**：
```
Error: java.lang.OutOfMemoryError: Java heap space
```

**原因**：
- Sentence Transformer模型较大（≈250MB）
- 批量处理大量文档（如：1000条 × 384维向量）

**调试方法**：
```java
// 1. 监控内存使用
Runtime runtime = Runtime.getRuntime();
System.out.println("Max Memory: " + runtime.maxMemory() / (1024 * 1024) + "MB");
System.out.println("Used Memory: " + (runtime.totalMemory() - runtime.freeMemory()) / (1024 * 1024) + "MB");

// 2. 检查模型加载
SentenceTransformer transformer = new SentenceTransformer();
System.out.println("Model loaded: " + transformer.getModelName());
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **批量处理** | 分批加载文档（如：每次100条） | 大规模知识库 |
| **模型优化** | 使用轻量级模型（如：all-MiniLM-L3-v2） | 内存受限环境 |
| **GPU加速** | 使用CUDA加速向量化 | 生产环境 |

**示例**：
```java
public class RAGRepository {
    private final SentenceTransformer sentenceTransformer;

    public RAGRepository() {
        // 使用轻量级模型（384维 → 256维）
        this.sentenceTransformer = new SentenceTransformer("all-MiniLM-L3-v2");
    }

    public void addDocumentsBatch(List<String> documents) {
        // 批量处理
        int batchSize = 100;
        for (int i = 0; i < documents.size(); i += batchSize) {
            List<String> batch = documents.subList(i, Math.min(i + batchSize, documents.size()));
            
            // 批量向量化
            List<Embedding> embeddings = sentenceTransformer.encode(batch);
            
            // 批量存储
            client.addEmbeddings("documents", 
                generateIds(batch.size()), 
                batch, 
                embeddings);
        }
    }
}
```

### 5.5.4 通用调试技巧

#### 5.5.4.1 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class AgentWithMemory {
    private static final Logger logger = LoggerFactory.getLogger(AgentWithMemory.class);

    public Response run(String input) {
        logger.info("Processing input: {}", input);
        
        try {
            // ... 业务逻辑
        } catch (Exception e) {
            logger.error("Failed to process input: {}", input, e);
            throw e;
        }
    }
}
```

**logback.xml配置**：
```xml
<configuration>
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <root level="debug">
        <appender-ref ref="STDOUT" />
    </root>
</configuration>
```

#### 5.5.4.2 单元测试（JUnit + Mockito）

**测试示例**：
```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;

public class ShortTermMemoryTest {
    @Test
    public void testAddMessage() {
        // 1. 创建Mock
        ShortTermMemory memory = new ShortTermMemory(20);

        // 2. 执行
        memory.add(new ShortTermMemory.Message("user", "Hello"));

        // 3. 验证
        List<ShortTermMemory.Message> recent = memory.getRecent();
        assertEquals(1, recent.size());
        assertEquals("user", recent.get(0).getRole());
        assertEquals("Hello", recent.get(0).getContent());
    }

    @Test
    public void testWindowOverflow() {
        ShortTermMemory memory = new ShortTermMemory(3); // 窗口3

        memory.add(new ShortTermMemory.Message("user", "1"));
        memory.add(new ShortTermMemory.Message("user", "2"));
        memory.add(new ShortTermMemory.Message("user", "3"));
        memory.add(new ShortTermMemory.Message("user", "4")); // 触发溢出

        List<ShortTermMemory.Message> recent = memory.getRecent();
        assertEquals(3, recent.size());
        assertEquals("2", recent.get(0).getContent()); // 最旧消息被移除
    }
}
```
