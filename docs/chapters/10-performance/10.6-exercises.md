## 10.6.2 代码改错（每题15分，共30分）

### 1. JMH基准测试改错

```java
// ❌ 错误代码（AgentBenchmark.java）
@State(Scope.Thread)
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MILLISECONDS)
public class AgentBenchmark {

    @Benchmark
    public String processQuery() {
        return "Result";
    }

    public static void main(String[] args) {
        // ❌ 未调用 JMH 主方法
        AgentBenchmark benchmark = new AgentBenchmark();
        benchmark.processQuery();
    }
}
```

**问题**：  
1. 未调用 `org.openjdk.jmh.Main.main(args)`  
2. 无法执行基准测试

**正确代码**：
```java
@State(Scope.Thread)
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MILLISECONDS)
@Warmup(iterations = 5, time = 1, timeUnit = TimeUnit.SECONDS)
@Measurement(iterations = 10, time = 1, timeUnit = TimeUnit.SECONDS)
@Fork(3)
public class AgentBenchmark {

    @Benchmark
    public String processQuery() {
        return "Result";
    }

    public static void main(String[] args) {
        org.openjdk.jmh.Main.main(args); // ✅ 调用 JMH 主方法
    }
}
```

### 2. 缓存未命中改错

```java
// ❌ 错误代码（CacheManager.java）
public class CacheManager {
    private final Cache<String, String> cache = Caffeine.newBuilder()
        .maximumSize(100) // ❌ 缓存容量太小
        .expireAfterWrite(1, TimeUnit.SECONDS) // ❌ 缓存过期时间太短
        .build();

    public String get(String key) {
        return cache.getIfPresent(key);
    }

    public void put(String key, String value) {
        cache.put(key, value);
    }
}
```

**问题**：  
1. 缓存容量太小（100）  
2. 缓存过期时间太短（1秒）

**正确代码**：
```java
public class CacheManager {
    private final Cache<String, String> cache = Caffeine.newBuilder()
        .maximumSize(100_000) // ✅ 增加缓存容量
        .expireAfterWrite(30, TimeUnit.MINUTES) // ✅ 延长TTL
        .recordStats() // ✅ 记录统计信息
        .build();

    public String get(String key) {
        return cache.getIfPresent(key);
    }

    public void put(String key, String value) {
        cache.put(key, value);
    }

    public CacheStats getStats() {
        return cache.stats();
    }
}
```

## 10.6.3 小型设计（30分）

### 设计任务：实现批量API调用

**需求**：  
设计一个`BatchAPI`类，实现批量处理多个查询。

**功能要求**：  
1. 分组处理（每组5个查询）  
2. 支持重试（最多3次）  
3. 方法签名：`List<String> processBatchQueries(List<String> queries)`

**参考答案**：
```java
public class BatchAPI {
    private static final int BATCH_SIZE = 5;
    private static final int MAX_RETRIES = 3;
    private final APIClient apiClient;

    public BatchAPI(APIClient apiClient) {
        this.apiClient = apiClient;
    }

    public List<String> processBatchQueries(List<String> queries) {
        List<List<String>> groups = Lists.partition(queries, BATCH_SIZE);

        List<String> results = new ArrayList<>();
        for (List<String> group : groups) {
            for (int i = 0; i < MAX_RETRIES; i++) {
                try {
                    results.addAll(apiClient.batchProcess(group));
                    break;
                } catch (TimeoutException e) {
                    if (i == MAX_RETRIES - 1) {
                        throw e;
                    }
                }
            }
        }

        return results;
    }
}
```

**背景**：

**问题**：

### 挑战2：Docker镜像优化

**背景**：  
当前Docker镜像体积过大（>500MB）。

**问题**：  
1. 如何使用多阶段构建减小镜像体积？  
2. 如何选择基础镜像？  
3. 如何排除调试依赖？

**参考答案**：

```dockerfile
## Dockerfile（多阶段构建）
## 第一阶段：构建
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /build
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests

## 第二阶段：运行
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=builder /build/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**优化策略**：
- 使用多阶段构建
- 选择`openjdk:17-jdk-slim`作为基础镜像
- 移除调试依赖（如`jmh-generator-annprocess`）
