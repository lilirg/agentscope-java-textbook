### 7.6.2 代码改错（每题15分，共30分）

#### 1. 自动配置类改错

```java
// ❌ 错误代码（AgentScopeAutoConfiguration.java）
@Configuration
public class AgentScopeAutoConfiguration {
    
    @Bean
    public Agent agent(String name, List<Tool> tools, Memory memory) {
        // ❌ 缺少@ConditionalOnMissingBean，会覆盖用户自定义Bean
        return new SimpleAgent(name, tools, memory);
    }
}
```

**问题**：  
1. 缺少`@ConditionalOnMissingBean`，会覆盖用户自定义Bean  
2. 缺少`@ConditionalOnProperty`，无法控制自动配置开关

**正确代码**：
```java
@Configuration
@ConditionalOnProperty(prefix = "agentscope", name = "enabled", havingValue = "true", matchIfMissing = true)
public class AgentScopeAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean(Agent.class)
    public Agent agent(String name, List<Tool> tools, Memory memory) {
        return new SimpleAgent(name, tools, memory);
    }
}
```

---

#### 2. AOP切面改错

```java
// ❌ 错误代码（AgentToolCallAspect.java）
@Aspect
public class AgentToolCallAspect {
    
    // ❌ 缺少@Component，无法注册为Spring Bean
    @Around("execution(* com.alibaba.agentscope.core.agent.Agent.execute(..))")
    public Object logToolCall(ProceedingJoinPoint joinPoint) throws Throwable {
        // ...
    }
}
```

**问题**：  
1. 缺少`@Component`，切面无法注册  
2. 切入点表达式错误（应为`Agent+`而非`Agent`）

**正确代码**：
```java
@Aspect

public class AgentToolCallAspect {
    
    @Around("execution(* com.alibaba.agentscope.core.agent.Agent+.execute(..))")  // ✅ 修改切入点
    public Object logToolCall(ProceedingJoinPoint joinPoint) throws Throwable {
        // ...
    }
}
```

### 7.6.3 小型设计（30分）

#### 设计任务：自定义Memory实现

**需求**：  
设计一个`RedisMemory`类，将记忆持久化到Redis。

**功能要求**：  
1. 继承`Memory`接口  
2. 使用Redis存储消息  
3. 支持自动过期（TTL）  

**参考答案**：
```java
package com.alibaba.agentscope.example.memory;

import com.alibaba.agentscope.core.memory.Memory;
import org.springframework.data.redis.core.RedisTemplate;

import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.TimeUnit;

public class RedisMemory implements Memory {
    private final RedisTemplate<String, String> redisTemplate;
    private final String key;
    private final int ttlSeconds;

    public RedisMemory(RedisTemplate<String, String> redisTemplate, String key, int ttlSeconds) {
        this.redisTemplate = redisTemplate;
        this.key = key;
        this.ttlSeconds = ttlSeconds;
    }

    @Override
    public void addMessage(String role, String content) {
        String message = String.format("%s: %s", role, content);
        redisTemplate.opsForList().rightPush(key, message);
        redisTemplate.expire(key, ttlSeconds, TimeUnit.SECONDS);
    }

    @Override
    public List<Message> getRecentMessages(int limit) {
        List<String> messages = redisTemplate.opsForList().range(key, 0, limit - 1);
        List<Message> result = new ArrayList<>();
        
        for (String msg : messages) {
            String[] parts = msg.split(": ", 2);
            if (parts.length == 2) {
                result.add(new Message(parts[0], parts[1]));
            }
        }
        
        return result;
    }

    @Override
    public void clear() {
        redisTemplate.delete(key);
    }

    @Override
    public void save() {
        // Redis已自动持久化，无需额外操作
    }

    @Override
    public void load() {
        // Redis自动加载，无需额外操作
    }

    // 内部类：消息定义
    public static class Message {
        private final String role;
        private final String content;

        public Message(String role, String content) {
            this.role = role;
            this.content = content;
        }

        public String getRole() { return role; }
        public String getContent() { return content; }
    }
}
```

// SimpleAgent.run()方法执行步骤：
// 1. 从Memory获取历史消息（getRecentMessages）
// 2. 构建提示词（Prompt），包含历史消息与当前输入

#### 挑战2：Spring Boot集成性能优化

**背景**：  
当前AgentScope-Java的自动配置每次启动都创建Agent Bean，但用户只在部分请求中使用。

**问题**：  
1. 如何实现Agent Bean的延迟加载（Lazy Loading）？  
2. 如何实现Agent Bean的缓存复用？  
3. 如何监控Agent的性能？

**参考答案**：

```java
// 1. 延迟加载
@Bean
@Lazy
public Agent agent(String name, List<Tool> tools, Memory memory) {
    return new SimpleAgent(name, tools, memory);
}

// 2. 缓存复用（单例模式已默认实现）

// 3. 性能监控
@Aspect
@Component
public class AgentPerformanceAspect {
    @Around("execution(* com.alibaba.agentscope.core.agent.Agent+.run(..))")
    public Object monitor(ProceedingJoinPoint joinPoint) throws Throwable {
        long startTime = System.currentTimeMillis();
        Object result = joinPoint.proceed();
        long endTime = System.currentTimeMillis();
        
        System.out.println("Agent.run()耗时: " + (endTime - startTime) + "ms");
        return result;
    }
}
```
