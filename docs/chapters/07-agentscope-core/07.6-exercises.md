### 7.6.2 代码改错（每题15分，共30分）

#### 1. 源码阅读改错

```java
// ❌ 错误代码（CustomAgent.java）
public class CustomAgent extends Agent {
    public Response run(String input) {
        // ❌ 忘记调用父类构造函数
        return new Response("Hello");
    }
}
```

**问题**：  
1. 未调用父类构造函数（`super(name)`）

**正确代码**：
```java
public class CustomAgent extends Agent {
    public CustomAgent(String name) {
        super(name);
    }
    
    public Response run(String input) {
        return new Response("Hello");
    }
}
```

---

#### 2. 性能分析改错

```java
// ❌ 错误代码（ThreadPoolConfig.java）
@Configuration
public class ThreadPoolConfig {
    @Bean
    public ThreadPoolTaskExecutor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(2);  // ❌ 线程池过小
        executor.setMaxPoolSize(5);   // ❌ 线程池过小
        executor.setQueueCapacity(1); // ❌ 队列过小
        executor.initialize();
        return executor;
    }
}
```

**问题**：  
1. 线程池配置过小，高并发时性能差

**正确代码**：
```java
@Configuration
public class ThreadPoolConfig {
    @Bean
    public ThreadPoolTaskExecutor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(20);  // ✅ 增加核心线程数
        executor.setMaxPoolSize(50);   // ✅ 增加最大线程数
        executor.setQueueCapacity(100);
        executor.initialize();
        return executor;
    }
}
```

### 7.6.3 小型设计（30分）

#### 设计任务：自定义Tool注册器

**需求**：  
设计一个`ToolRegistry`类，注册多个自定义Tool。

**功能要求**：  
1. 支持添加自定义Tool  
2. 获取所有Tool列表  
3. 根据名称查找Tool  

**参考答案**：
```java
public class ToolRegistry {
    private final Map<String, Tool> tools = new ConcurrentHashMap<>();

    /**
     * 添加工具
     * @param tool 工具
     */
    public void addTool(Tool tool) {
        tools.put(tool.getName(), tool);
    }

    /**
     * 获取所有工具
     * @return 工具列表
     */
    public List<Tool> getAllTools() {
        return new ArrayList<>(tools.values());
    }

    /**
     * 根据名称查找工具
     * @param name 工具名称
     * @return 工具（不存在返回null）
     */
    public Tool getToolByName(String name) {
        return tools.get(name);
    }
}
```

#### 挑战2：源码阅读与PR

**背景**：  
发现AgentScope-Java的`Agent.run()`方法缺少异常处理，建议优化。

**问题**：  
1. 阅读源码`Agent.java`  
2. 提出改进方案  
3. 编写PR描述

**参考答案**：

```markdown
# PR描述

## 问题描述
`Agent.run()`方法缺少异常处理，导致Agent崩溃时无法返回友好错误信息。

## 改进方案
添加try-catch块，记录异常并返回友好错误。

## 代码示例
```java
@Override
public Response run(String input) {
    try {
        // ... 业务逻辑
        return response;
    } catch (Exception e) {
        logger.error("Agent执行失败", e);
        return new Response("发生错误，请稍后重试", Collections.singletonMap("error", e.getMessage()));
    }
}
```

## 测试
- 单元测试：验证异常处理
- 集成测试：验证友好错误信息
```
