## 源码编译相关陷阱

### 陷阱1：源码编译失败

**现象**：
```
Error: Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.10.1:compile
> Cannot find symbol
  symbol:   class Agent
  location: package com.alibaba.agentscope.core.agent
```

**原因**：
- 源码版本与JDK版本不匹配（如：源码要求JDK 17，但安装了JDK 11）
- Maven依赖未下载完成

**调试方法**：
```bash
# 1. 检查JDK版本
mvn -version

# 2. 检查pom.xml中的JDK配置
cat pom.xml | grep maven.compiler

# 3. 清理并重新下载依赖
mvn clean install -U
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **升级JDK** | 安装JDK 17+ | 源码要求高版本 |
| **降级源码** | 切换到v0.1.0版本（支持JDK 11） | 无法升级JDK |
| **网络代理** | 配置Maven代理 | 国内网络受限 |

---

### 陷阱2：源码导入IDE失败

**现象**：
```
Error: Cannot resolve symbol 'Agent'
Error: Cannot resolve symbol 'Tool'
```

**原因**：
- IDE未正确识别Maven项目
- 源码依赖未下载

**调试方法**：
```bash
# 1. 在IDEA中执行
# File → Invalidate Caches and Restart → Invalidate and Restart

# 2. 重新导入Maven项目
# 右键pom.xml → Maven → Reimport
```

**解决方案**：
```bash
# 生成IDEA项目文件
mvn idea:idea

# 或生成Eclipse项目文件
mvn eclipse:eclipse

# 然后导入IDE
```

## 配置加载相关陷阱

### 陷阱3：配置未生效

**现象**：
```
Warning: No configuration found for 'agentscope.agent.name'
Using default value: 'DefaultAgent'
```

**原因**：
- 配置文件路径错误（应为`src/main/resources/application.yml`）
- 配置前缀错误（应为`agentscope.*`而非`agent.*`）

**调试方法**：
```java
// 1. 打印配置
@Autowired
private AgentScopeProperties properties;

@PostConstruct
public void printConfig() {
    System.out.println("Agent name: " + properties.getAgent().getName());
    System.out.println("OpenAI API Key: " + properties.getOpenai().getApiKey());
}

// 2. 检查配置文件路径
// 确保文件位于：src/main/resources/application.yml
```

**解决方案**：
```yaml
# application.yml（正确位置：src/main/resources/）
agentscope:
  agent:
    name: MyAgent  # ✅ 正确前缀
  openai:
    api-key: ${OPENAI_API_KEY}
```

### 陷阱4：环境变量未加载

**现象**：
```
Error: OPENAI_API_KEY is not set
```

**原因**：
- `.env`文件未加载（Spring Boot不自动加载`.env`）
- 环境变量未设置（Windows/Linux/Mac路径不同）

**调试方法**：
```bash
# 1. 检查环境变量
# Windows
echo %OPENAI_API_KEY%
# Linux/Mac
echo $OPENAI_API_KEY

# 2. 检查application.yml
# 应使用 ${OPENAI_API_KEY} 引用环境变量
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **使用System.getenv()** | 在代码中直接读取环境变量 | 生产环境 |
| **使用dotenv-java** | 加载`.env`文件 | 开发环境 |
| **使用application.yml** | 直接写死配置（不推荐） | 临时测试 |

**示例**：
```java
// 使用dotenv-java加载.env
Dotenv dotenv = Dotenv.configure()
    .directory(".env")
    .ignoreIfMissing()
    .load();
String apiKey = dotenv.get("OPENAI_API_KEY");
```

## Spring Boot集成相关陷阱

### 陷阱5：Agent Bean未创建

**现象**：
```
Error: No qualifying bean of type 'Agent' available
```

**原因**：
- 自动配置未启用（`agentscope.enabled=false`）
- 用户自定义了Agent Bean，覆盖了自动配置

**调试方法**：
```bash
# 1. 启动时打印Bean信息
java -jar app.jar --debug

# 2. 检查自动配置
# 应包含：AgentScopeAutoConfiguration
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **启用自动配置** | 设置`agentscope.enabled=true` | 默认配置 |
| **排除自动配置** | `@SpringBootApplication(exclude = AgentScopeAutoConfiguration.class)` | 自定义配置 |
| **合并Bean** | 实现`BeanFactoryPostProcessor` | 高级扩展 |

---

### 陷阱6：AOP代理未生效

**现象**：
```
Warning: AgentToolCallAspect not applied
```

**原因**：
- AOP切面未注册（缺少`@Component`或`@Configuration`）
- 切入点表达式错误

**调试方法**：
```java
// 1. 检查切面类
@Component  // ✅ 必须有
@Aspect
public class AgentToolCallAspect {
    // ...
}

// 2. 启动时启用AOP调试
java -jar app.jar --debug | grep "Aspect"
```

**解决方案**：
```java
// 正确的切入点表达式
@Around("execution(* com.alibaba.agentscope.core.agent.Agent+.execute(..))")
public Object logToolCall(ProceedingJoinPoint joinPoint) throws Throwable {
    // ...
}
```

## 通用调试技巧

### 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class AgentScopeAutoConfiguration {
    private static final Logger logger = LoggerFactory.getLogger(AgentScopeAutoConfiguration.class);

    @Bean
    public Agent agent(String name, List<Tool> tools, Memory memory) {
        logger.info("Creating Agent: {} with {} tools", name, tools.size());
        
        try {
            return new SimpleAgent(name, tools, memory);
        } catch (Exception e) {
            logger.error("Failed to create Agent", e);
            throw e;
        }
    }
}
```

### 单元测试（JUnit + Mockito）

**测试示例**：
```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;
import static org.junit.jupiter.api.Assertions.*;

public class AgentScopeAutoConfigurationTest {
    
    @Test
    public void testAgentBeanCreation() {
        // 1. 创建Mock
        List<Tool> tools = Mockito.mock(List.class);
        Memory memory = Mockito.mock(Memory.class);
        
        // 2. 执行
        Agent agent = new SimpleAgent("TestAgent", tools, memory);
        
        // 3. 验证
        assertEquals("TestAgent", agent.getName());
        assertNotNull(agent);
    }
}
```
