### 11.2.2 容器化与部署工具

#### 1. Docker（容器化）

| 版本 | 获取路径 | 说明 |
|------|---------|------|
| Docker ≥24 | [Docker官网](https://www.docker.com/get-started/) | **推荐版本** |

**Dockerfile优化**：
```dockerfile
# Dockerfile（多阶段构建）
# 第一阶段：构建
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /build
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests

# 第二阶段：运行
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=builder /build/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**构建与运行**：
```bash
# 1. 构建镜像
docker build -t agent-app:latest .

# 2. 运行容器
docker run -p 8080:8080 agent-app:latest
```

| 版本 | 获取路径 | 说明 |
|------|---------|------|
| Kubernetes ≥1.28 | [K8s官网](https://kubernetes.io/docs/setup/) | **推荐版本** |

**K8s配置示例**（Deployment）：
```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: agent-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: agent-app
  template:
    metadata:
      labels:
        app: agent-app
    spec:
      containers:
      - name: agent-app
        image: agent-app:latest
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "256Mi"
            cpu: "100m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /actuator/health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5
        readinessProbe:
          httpGet:
            path: /actuator/ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 3
```

### 11.2.4 告警系统工具

#### 1. 钉钉机器人

| 获取路径 | 说明 |
|---------|------|
| [钉钉开放平台](https://open.dingtalk.com/) | 创建钉钉机器人 |

**钉钉机器人配置**：
```java
@Service
public class DingTalkNotifier {
    private static final String WEBHOOK_URL = System.getenv("DINGTALK_WEBHOOK_URL");
    private static final Logger logger = LoggerFactory.getLogger(DingTalkNotifier.class);
    
    public void notify(String title, String message) {
        if (WEBHOOK_URL == null || WEBHOOK_URL.isEmpty()) {
            logger.warn("DingTalk webhook URL not configured");
            return;
        }
        
        Map<String, Object> payload = new HashMap<>();
        payload.put("msgtype", "text");
        Map<String, String> text = new HashMap<>();
        text.put("content", String.format("%s: %s", title, message));
        payload.put("text", text);
        
        try {
            restTemplate.postForObject(WEBHOOK_URL, payload, String.class);
        } catch (Exception e) {
            logger.error("Failed to send DingTalk notification", e);
        }
    }
}
```

#### 2. 邮件告警

| 库 | Maven坐标 | 说明 |
|----|-----------|------|
| JavaMail | `com.sun.mail:javax.mail:1.6.2` | 邮件发送库 |

**邮件告警配置**：
```java
@Service
public class EmailNotifier {
    private static final String SMTP_HOST = System.getenv("SMTP_HOST");
    private static final String SMTP_PORT = System.getenv("SMTP_PORT");
    private static final String SMTP_USER = System.getenv("SMTP_USER");
    private static final String SMTP_PASSWORD = System.getenv("SMTP_PASSWORD");
    private static final String FROM_EMAIL = System.getenv("FROM_EMAIL");
    private static final Logger logger = LoggerFactory.getLogger(EmailNotifier.class);
    
    public void notify(String to, String subject, String body) {
        Properties props = new Properties();
        props.put("mail.smtp.host", SMTP_HOST);
        props.put("mail.smtp.port", SMTP_PORT);
        props.put("mail.smtp.auth", "true");
        props.put("mail.smtp.starttls.enable", "true");
        
        Session session = Session.getInstance(props, new Authenticator() {
            @Override
            protected PasswordAuthentication getPasswordAuthentication() {
                return new PasswordAuthentication(SMTP_USER, SMTP_PASSWORD);
            }
        });
        
        try {
            Message message = new MimeMessage(session);
            message.setFrom(new InternetAddress(FROM_EMAIL));
            message.setRecipients(Message.RecipientType.TO, InternetAddress.parse(to));
            message.setSubject(subject);
            message.setText(body);
            
            Transport.send(message);
        } catch (Exception e) {
            logger.error("Failed to send email", e);
        }
    }
}
```

### 11.2.5 日志分析工具

#### 1. ELK（Elasticsearch + Logstash + Kibana）

| 组件 | 获取路径 | 说明 |
|------|---------|------|
| Elasticsearch | [elastic.co](https://www.elastic.co/downloads/elasticsearch) | 日志存储与检索 |
| Logstash | [elastic.co](https://www.elastic.co/downloads/logstash) | 日志收集与处理 |
| Kibana | [elastic.co](https://www.elastic.co/downloads/kibana) | 日志可视化 |

**ELK架构**：
```
应用日志 → Logstash → Elasticsearch → Kibana
```

**Logstash配置**：
```ruby
# logstash.conf
input {
  file {
    path => "/var/log/app/*.log"
    start_position => "beginning"
  }
}

filter {
  grok {
    match => { "message" => "%{TIMESTAMP_ISO8601:timestamp} %{LOGLEVEL:level} %{JAVACLASS:class} - %{GREEDYDATA:message}" }
  }
}

output {
  elasticsearch {
    hosts => ["http://localhost:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
  }
}
```
