3. 提示词优化原则是：______、______、______  
4. 提示词评估指标是：______、______、______  
5. 提示词评估方法是：______、______、______  
6. 开发环境必需工具是：______、______、______、______

2. 明确指令、提供上下文、提供示例

## 小型设计（30分）

**说明**：每题15分，共2题，满分30分

### 题目1：提示词设计（15分）

**任务**：设计“生成代码”提示词，要求：

1. 指令明确  
2. 上下文完整  
3. 示例充分  

**评分标准**：
- 指令明确（5分）  
- 上下文完整（5分）  
- 示例充分（5分）  

**参考答案**：
```
你是一个Java开发助手，请生成高质量的Java代码。

上下文：
- 项目：Spring Boot 3.2.5
- 技术栈：Java 17
- 需求：实现一个简单的REST API

示例：
Q: 实现一个查询用户信息的API
A: public class UserController {
       @GetMapping("/users/{id}")
       public User getUser(@PathVariable Long id) {
           return userService.findById(id);
       }
   }
```

---

### 题目2：提示词评估指标设计（15分）

**任务**：设计“生成文本”提示词评估指标，要求：

1. 准确性评估  
2. 完整性评估  
3. 效率评估  

**评分标准**：
- 准确性评估合理（5分）  
- 完整性评估合理（5分）  
- 效率评估合理（5分）  

**参考答案**：
```
评估指标：
1. 准确性：匹配度（字符串匹配、语义匹配）
2. 完整性：长度（字符数、词数）、信息完整性
3. 效率：生成速度（毫秒/字符）
```
