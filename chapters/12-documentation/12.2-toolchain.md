### 12.2.5 CI/CD工具

#### 1. GitHub Actions

| 特点 | 说明 |
|------|------|
| **免费**：GitHub官方CI/CD |
| **集成**：与GitHub深度集成 |
| **灵活**：YAML配置 |

**GitHub Actions配置**：
```yaml
# .github/workflows/ci.yml
name: CI/CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
        distribution: 'temurin'
    
    - name: Build with Maven
      run: mvn clean package
    
    - name: Run tests
      run: mvn test
    
    - name: Build Docker image
      run: docker build -t agent-app:latest .
    
    - name: Push to Docker Hub
      run: |
        docker login -u ${{ secrets.DOCKER_USERNAME }} -p ${{ secrets.DOCKER_PASSWORD }}
        docker push agent-app:latest
```

#### 2. Jenkins

| 特点 | 说明 |
|------|------|
| **强大**：插件丰富 |
| **灵活**：Groovy脚本 |
| **自建**：需要自行部署 |

**Jenkins Pipeline示例**：
```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'mvn clean package'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Build Docker') {
            steps {
                sh 'docker build -t agent-app:latest .'
            }
        }
        stage('Deploy') {
            steps {
                sh 'docker push agent-app:latest'
            }
        }
    }
}
```

#### 2. API文档规范

| 部分 | 说明 |
|------|------|
| **接口描述**：接口功能 |
| **请求参数**：参数说明 |
| **响应结果**：返回格式 |
| **错误码**：错误处理 |

**API文档示例**：
```markdown
## Agent API

**接口地址**：`GET /api/agent/status`

| agentId | string | 是 | Agent ID |

{
  "code": 200,
  "message": "success",
  "data": {
    "status": "running",
    "cpuUsage": 0.5,

**错误码**：
| 错误码 | 说明 |
|--------|------|
| 400 | 请求参数错误 |
