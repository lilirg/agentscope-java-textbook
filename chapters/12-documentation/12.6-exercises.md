#### 2. Git工作流改错

```bash
# ❌ 错误代码（Git工作流）
git checkout main
git checkout -b feature/add-agent-api
# 开发完成后...
git commit -m "add feature"
git push origin feature/add-agent-api
# 直接合并到main ❌

```

**问题**：  
1. 直接合并到main，缺少PR审查

**正确代码**：
```bash
# ✅ 正确代码（Git工作流）
git checkout main
git checkout -b feature/add-agent-api
# 开发完成后...
git add .
git commit -m "feat: add agent API"  # ✅ 使用Conventional Commits
git push origin feature/add-agent-api
# ✅ 开启PR，等待审查通过后合并
```

### 12.6.4 拓展挑战（20分）

#### 挑战1：CI/CD流程配置

**背景**：  
当前项目缺少CI/CD流程，需要配置GitHub Actions。

**问题**：  
1. 如何配置持续集成（CI）流程？  
2. 如何配置持续部署（CD）流程？  
3. 如何配置Docker镜像构建与推送？

**参考答案**：

```yaml
# .github/workflows/ci.yml
name: CI/CD

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
        distribution: 'temurin'
    
    - name: Build with Maven
      run: mvn clean package
    
    - name: Run tests
      run: mvn test
    
    - name: Check style
      run: mvn checkstyle:check
    
    - name: Build Docker image
      run: docker build -t agent-app:latest .
```

```yaml
# .github/workflows/cd.yml
name: CD

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up JDK 17
      uses: actions/setup-java@v3
      with:
        java-version: '17'
        distribution: 'temurin'
    
    - name: Build with Maven
      run: mvn clean package
    
    - name: Run tests
      run: mvn test
    
    - name: Build Docker image
      run: docker build -t agent-app:latest .
    
    - name: Push to Docker Hub
      run: |
        docker login -u ${{ secrets.DOCKER_USERNAME }} -p ${{ secrets.DOCKER_PASSWORD }}
        docker push agent-app:latest
    
    - name: Deploy to Kubernetes
      run: |
        kubectl apply -f kubernetes/deployment.yaml
        kubectl apply -f kubernetes/service.yaml
```

**CI/CD配置要点**：
- CI流程：构建、测试、检查、构建Docker镜像
- CD流程：推送Docker镜像到仓库、部署到K8s
- Docker配置：登录Docker Hub、推送镜像、K8s部署
