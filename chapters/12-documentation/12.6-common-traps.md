### 12.5.1 Agent文档化相关陷阱

#### 12.5.1.1 陷阱1：README缺失

**现象**：
```
Warning: README not found, new members cannot understand the project
```

**原因**：
- 项目缺少README文件
- README内容不完整

**调试方法**：
```bash
# 1. 检查README是否存在
ls -la | grep README.md

# 2. 检查README内容
cat README.md
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **创建README** | 创建完整的README文档 | 新项目 |
| **完善README** | 补充缺失内容 | 旧项目 |

**示例**：
```markdown
# AgentScope-Java

> 一个轻量级的Java Agent框架

## 🚀 特性
- 轻量级：无依赖，最小化设计
- 易用：简单API，快速上手
- 扩展：插件化架构，易于扩展

## 📦 安装
```bash
# 1. 克隆项目
git clone https://github.com/lilirg/agentscope-java.git

# 2. 编译项目
mvn clean package

# 3. 运行示例
java -jar target/agentscope-java-example.jar
```

## 💡 使用
- 查看API文档：docs/API.md
- 查看架构设计：docs/ARCHITECTURE.md

## 🤝 贡献
欢迎提交Issue和PR！

## 📄 许可证
[MIT License](LICENSE)
```

#### 12.5.1.2 陷阱2：API文档不完整

**现象**：
```
Warning: API documentation is incomplete, developers cannot use the API
```

**原因**：
- API文档缺失
- API文档未及时更新

**调试方法**：
```bash
# 1. 检查API文档
ls docs/API.md

# 2. 检查API文档内容
cat docs/API.md
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **生成API文档** | 使用Swagger生成API文档 | REST API |
| **手动编写API文档** | 手动编写API文档 | 自定义API |
| **定期更新API文档** | 每次API变更更新文档 | 维护阶段 |

**示例**：
```markdown
# AgentScope-Java API文档

## Agent API

### 1. 查询Agent状态

**接口地址**：`GET /api/agent/status`

**请求参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| agentId | string | 是 | Agent ID |

**响应结果**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": "running",
    "cpuUsage": 0.5,
    "memoryUsage": 0.6
  }
}
```

**错误码**：
| 错误码 | 说明 |
|--------|------|
| 400 | 请求参数错误 |
| 404 | Agent不存在 |
| 500 | 服务器错误 |
```

### 12.5.2 协作规范相关陷阱

#### 12.5.2.1 陷阱3：Git工作流混乱

**现象**：
```
Warning: Git workflow is混乱, code conflicts are frequent
```

**原因**：
- 多人在main分支开发
- 无代码审查
- 提交信息不规范

**调试方法**：
```bash
# 1. 检查分支情况
git branch -a

# 2. 检查提交历史
git log --oneline --graph

# 3. 检查PR情况
git log --oneline --grep="Merge pull request"
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **使用Feature Branch** | 所有功能在分支开发 | 通用 |
| **强制PR审查** | 所有代码合并必须通过PR | 通用 |
| **使用Conventional Commits** | 规范提交信息 | 通用 |

**示例**：
```bash
# 1. 创建Feature分支
git checkout -b feature/add-agent-api

# 2. 开发完成后，提交PR
git push origin feature/add-agent-api
```

**现象**：

**原因**：

```bash

git log --oneline --grep="Merge pull request" | wc -l

# 2. 检查PR审查
gh pr list --limit 100

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **强制PR** | 所有代码合并必须通过PR | 通用 |
| **审查清单** | 使用代码审查清单 | 通用 |
| **审查反馈** | 及时反馈审查意见 | 通用 |

**示例**：
```markdown
## 📝 PR描述

**功能描述**：  
[功能描述]

**相关Issue**：  
[Issue链接]

**测试说明**：  
- [单元测试]
- [集成测试]

**注意事项**：  
[注意事项]
```

### 12.5.3 项目管理相关陷阱

#### 12.5.3.1 陷阱5：Jira任务管理混乱

**现象**：
```
Warning: Jira tasks are混乱, progress is unclear
```

**原因**：
- 任务分配不明确
- 任务状态不更新
- 问题无跟踪

**调试方法**：
```bash
# 1. 检查任务数量
jira issue list --assignee current

# 2. 检查任务状态
jira issue list --status "In Progress"
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **明确任务分配** | 每个任务明确负责人 | Scrum |
| **更新任务状态** | 每日更新任务状态 | Scrum |
| **问题跟踪** | 使用Jira Issue跟踪问题 | 通用 |

**示例**：
```markdown
## 🐛 Bug描述

**类型**：Bug

**优先级**：Critical / High / Medium / Low

**描述**：  
[描述]

**重现步骤**：  
1. [步骤1]
2. [步骤2]
3. [步骤3]

**预期结果**：  
[预期结果]

**实际结果**：  
[实际结果]

**环境**：  
- OS: [操作系统]
- Java: [Java版本]
- AgentScope: [版本]

**附件**：  
- [截图]
- [日志]
```

### 12.5.4 CI/CD相关陷阱

#### 12.5.4.1 陷阱6：CI/CD失败

**现象**：
```
Error: CI/CD pipeline failed
```

**原因**：
- 构建失败
- 测试失败
- 部署失败

**调试方法**：
```bash
# 1. 检查CI/CD日志
gh pr checks <pr-number>

# 2. 检查构建日志
cat logs/build.log

# 3. 检查测试日志
cat logs/test.log
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **构建失败** | 检查Maven配置、依赖版本 | 构建失败 |
| **测试失败** | 检查测试用例、测试环境 | 测试失败 |
| **部署失败** | 检查Docker配置、K8s配置 | 部署失败 |

**示例**：
```yaml
# ci.yml
- name: Build with Maven
  run: mvn clean package
  
- name: Run tests
  run: mvn test
  
- name: Check style
  run: mvn checkstyle:check
```

### 12.5.5 通用调试技巧

#### 12.5.5.1 单元测试（JUnit + Mockito）

**测试示例**（AgentService）：
```java
import org.junit.jupiter.api.Test;
import org.mockito.Mockito;

import static org.junit.jupiter.api.Assertions.*;
import static org.mockito.Mockito.*;

public class AgentServiceTest {
    
    @Test
    public void testProcessQuery() {
        // 1. 创建Mock
        AgentClient agentClient = Mockito.mock(AgentClient.class);
        CacheManager cacheManager = Mockito.mock(CacheManager.class);
        PerformanceMonitor performanceMonitor = Mockito.mock(PerformanceMonitor.class);
        
        AgentService agentService = new AgentService(agentClient, cacheManager, performanceMonitor);
        
        // 2. 模拟
        when(agentClient.call("test")).thenReturn("Result: test");
        when(cacheManager.get("test")).thenReturn(null);
        
        // 3. 执行
        String result = agentService.processQuery("test");
        
        // 4. 验证
        assertEquals("Result: test", result);
        verify(agentClient, times(1)).call("test");
        verify(cacheManager, times(1)).put("test", "Result: test");
    }
}
```

import org.slf4j.Logger;

public class AgentService {
    private static final Logger logger = LoggerFactory.getLogger(AgentService.class);
    
    public String processQuery(String query) {
        logger.debug("Processing query: {}", query);
        
        try {
            // 业务逻辑
            return result;
        } catch (Exception e) {
            logger.error("Failed to process query: {}", query, e);
            throw e;
        }
    }
}
