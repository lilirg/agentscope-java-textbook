## 🚨 消息队列选型指南（必须遵守）

### 选型对比

| 特性 | Kafka | RabbitMQ | Redis Streams |
|------|-------|----------|---------------|
| **吞吐量** | 高（100K+ msg/s） | 中（10K msg/s） | 高（50K+ msg/s） |
| **延迟** | 中（10-50ms） | 低（1-10ms） | 低（1-5ms） |
| **持久化** | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| **分布式** | ✅ 原生支持 | ✅ 支持 | ❌ 依赖Redis Cluster |
| **易用性** | ❌ 较复杂 | ✅ 简单 | ✅ 简单 |
| **适用场景** | 大数据处理、日志收集 | 传统消息队列 | 快速开发、小规模应用 |

### 选型建议

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| **高并发客服系统** | Kafka | 吞吐量高，支持1000+并发 |
| **学习/演示环境** | RabbitMQ | 易用性高，管理界面友好 |
| **快速原型开发** | Redis Streams | 轻量级，无需额外部署 |
| **分布式日志收集** | Kafka | 原生支持分布式，高吞吐量 |

## 快速开始脚本（Windows/Linux）

```bash
## setup.sh / setup.bat（根据系统选择）
## 安装JDK 17（使用sdkman）
sdk install java 17-tem

## 安装Maven（Linux）
sudo apt-get install maven

## 启动Kafka（Docker）
docker run -p 9092:9092 -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 confluentinc/cp-kafka:7.5.0

## 启动RabbitMQ（Docker）
docker run -p 15672:15672 -p 5672:5672 rabbitmq:3-management

## 启动Redis（Docker）
docker run -p 6379:6379 redis:7-alpine

## 初始化项目（Maven）
mvn archetype:generate \
    -DgroupId=com.textbook.chapter8 \
    -DartifactId=multi-agent-system \
    -DarchetypeArtifactId=maven-archetype-quickstart \
    -DinteractiveMode=false
```

## 本节思考题

**问题1**：如何选择消息队列（Kafka vs RabbitMQ vs Redis Streams）？  
**提示**：对比吞吐量、延迟、持久化、易用性四个维度  

**问题2**：为什么Kafka适合高并发场景，而RabbitMQ适合学习环境？  
**提示**：思考Kafka的分布式架构与RabbitMQ的简单设计差异  

**问题3**：如何验证消息队列是否正常工作？  
**提示**：思考生产者/消费者测试、管理界面检查、日志分析
