## 代码改错（30分）

**说明**：每题15分，共2题，满分30分

### 题目1：Agent需求文档缺失（15分）

**问题**：以下是某Agent项目的需求文档，请找出其中的5个问题并修改：

```markdown
# Agent需求文档

# 1. 项目背景
项目名称：Agent系统
项目目标：实现一个Agent系统
项目范围：开发

# 2. 用户需求
用户角色：管理员、用户
用户故事：
- 管理员登录
- 用户查询Agent

# 3. 功能需求
功能列表：
- 功能1：管理员登录
- 功能2：用户查询Agent

# 4. 验收标准
- 需求文档完整
- 系统实现完整
```

**改错要求**：
1. 找出5个问题  
2. 修改问题，完善需求文档  
3. 每个问题修改得当得3分，满分15分

**参考答案**：
```
问题1：用户故事缺少验收标准  
修改：补充用户故事的验收标准

问题2：功能需求缺少优先级  
修改：添加功能优先级

问题3：缺少非功能需求  
修改：添加性能需求、安全需求、可用性需求

问题4：缺少技术需求  
修改：添加技术栈

问题5：缺少项目约束  
修改：添加时间约束、成本约束、资源约束
```

## 小型设计（30分）

**说明**：每题15分，共2题，满分30分

### 题目1：Agent用户故事设计（15分）

**任务**：为“查询天气”功能设计Agent用户故事，要求：

1. 用户角色：用户  
2. 功能描述：查询天气  
3. 验收标准：≥3条  

**评分标准**：
- 用户角色明确（3分）  
- 功能描述清晰（3分）  
- 验收标准完整（9分）  

**参考答案**：
```markdown
用户故事：查询天气

作为 用户，
我希望 查询天气，
以便 了解天气情况。

验收标准：
1. 用户可以输入城市名称查询天气
2. 查询成功时，返回天气信息
3. 查询失败时，显示错误信息
```

## 拓展挑战（20分）

**说明**：每题10分，共2题，满分20分

### 题目1：Agent需求变更处理（10分）

**任务**：某Agent项目在开发中期，客户提出新需求，要求：

1. 需求变更评估  
2. 变更流程设计  
3. 影响分析  

**评分标准**：
- 需求变更评估合理（3分）  
- 变更流程完整（4分）  
- 影响分析全面（3分）  

**参考答案**：
```
1. 需求变更评估：
- 影响范围：新增功能
- 工作量评估：+5人日
- 风险评估：中

2. 变更流程：
- 客户提出变更需求
- 变更影响分析
- 变更评审
- 变更批准
- 需求文档更新
- 项目计划更新

3. 影响分析：
- 时间影响：+5天
- 成本影响：+¥1万
- 资源影响：+1人
```

---

### 题目2：API调用重试策略设计（10分）

**任务**：设计API调用重试策略，要求：

1. 重试次数  
2. 重试间隔  
3. 错误处理  

**评分标准**：
- 重试次数合理（3分）  
- 重试间隔合理（3分）  
- 错误处理完整（4分）  

**参考答案**：
```
重试策略：
1. 重试次数：3次
2. 重试间隔：指数退避（1s, 2s, 4s）
3. 错误处理：
   - 4xx错误：不重试，直接返回错误
   - 5xx错误：重试3次
   - 超时错误：重试3次
```
