### 7.5.1 源码阅读相关陷阱

#### 7.5.1.1 陷阱1：忽略版本兼容性

**现象**：
```
Error: java.lang.NoSuchMethodError: 
com.alibaba.agentscope.core.agent.Agent.run(Lcom/textbook/chapter6/MultimodalInput;)...
```

**原因**：
- AgentScope-Java v0.1.0与v0.2.0 API不兼容
- 未阅读CHANGELOG.md

**调试方法**：
```bash
# 1. 检查依赖版本
mvn dependency:tree | grep agentscope-java

# 2. 阅读CHANGELOG.md
# 访问 https://github.com/modelscope/agentscope-java/blob/main/CHANGELOG.md
```

**解决方案**：
```xml
<!-- pom.xml -->
<dependency>
    <groupId>com.alibaba.agentscope</groupId>
    <artifactId>agentscope-spring</artifactId>
    <version>0.2.0</version>
    <!-- 确保版本一致 -->
</dependency>
```

#### 7.5.1.2 陷阱2：误读源码逻辑

**现象**：
```
错误理解：Tool.execute()接收String参数
实际源码：Tool.execute()接收Map<String, Object>参数
```

**原因**：
- 未仔细阅读Javadoc
- 仅凭方法名猜测参数类型

**调试方法**：
```java
// 1. 阅读Javadoc
// @param parameters 工具参数（Map类型）

// 2. 查看测试用例
// test/ToolTest.java

// 3. 使用IDE跳转到源码
// Ctrl+B（Windows）或 Cmd+B（Mac）
```

**解决方案**：
```java
// 正确调用方式
Tool tool = new CustomTool("test", "测试工具");
Map<String, Object> params = new HashMap<>();
params.put("query", "hello");
String result = tool.execute(params);
```

### 7.5.2 调试相关陷阱

#### 7.5.2.1 陷阱3：调试工具配置错误

**现象**：
```
Error: JDWP: Unable to get JNI 1.2 environment
```

**原因**：
- JVM参数配置错误
- JDWP调试端口冲突

**调试方法**：
```bash
# 1. 检查JDWP参数
java -agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=5005

# 2. 检查端口占用
netstat -ano | findstr :5005
lsof -i :5005
```

**解决方案**：
```bash
# 正确的JDWP参数
-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005
```

### 7.5.3 性能优化相关陷阱

#### 7.5.3.1 陷阱5：线程池配置不当

**现象**：
```
高并发下Agent响应时间从500ms激增到3s
```

**原因**：
- 线程池大小固定（默认10）
- 高并发时线程争用

**调试方法**：
```java
// 1. 监控线程池状态
ThreadPoolTaskExecutor executor = (ThreadPoolTaskExecutor) applicationContext.getBean("taskExecutor");
System.out.println("活跃线程数：" + executor.getActiveCount());
System.out.println("队列大小：" + executor.getQueue().size());
```

**解决方案**：
```java
@Configuration
public class ThreadPoolConfig {
    @Bean
    public ThreadPoolTaskExecutor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(20);  // 增加核心线程数
        executor.setMaxPoolSize(50);   // 增加最大线程数
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("Agent-");
        executor.initialize();
        return executor;
    }
}
```

#### 7.5.3.2 陷阱6：缓存策略失效

**现象**：
```
LLM API调用次数激增，成本上升
```

**原因**：
- 未实现缓存（如：Redis）
- 相同输入重复调用LLM

**调试方法**：
```java
// 1. 记录API调用日志
logger.info("LLM API调用：{}", prompt);

// 2. 监控调用频率
// 统计每分钟调用次数
```

**解决方案**：
```java
@Service
public class CachedLLMService implements LLMService {
    private final RedisTemplate<String, String> redisTemplate;
    private final LLMService llmService;

    @Override
    public String generate(String prompt) {
        // 1. 检查缓存
        String cached = redisTemplate.opsForValue().get("llm:" + prompt.hashCode());
        if (cached != null) {
            return cached;
        }

        // 2. 调用LLM API
        String result = llmService.generate(prompt);

        // 3. 缓存结果
        redisTemplate.opsForValue().set("llm:" + prompt.hashCode(), result, 1, TimeUnit.HOURS);

        return result;
    }
}
```

### 7.5.4 通用调试技巧

#### 7.5.4.1 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class AgentService {
    private static final Logger logger = LoggerFactory.getLogger(AgentService.class);

    public String execute(String input) {
        logger.info("Processing input: {}", input);
        
        try {
            // ... 业务逻辑
        } catch (Exception e) {
            logger.error("Failed to process input: {}", input, e);
            throw e;
        }
    }
}
```
