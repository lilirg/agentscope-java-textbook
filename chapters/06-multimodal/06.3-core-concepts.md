### 6.3.1 图片输入处理（CLIP模型）

#### 6.3.1.1 概念定义

**CLIP**（Contrastive Language–Image Pretraining）：一种联合嵌入模型，将图片和文本映射到同一向量空间。

**设计目标**：
- ✅ 图片→向量（512维/1024维）
- ✅ 文本→向量（与图片同维度）
- ✅ 余弦相似度计算图文匹配度

#### 6.3.1.2 技术原理

**训练方式**：
```
输入：(图片, 文本)对（30亿对）
目标：最大化匹配对相似度，最小化非匹配对相似度

示例：
图片："一只猫在沙发上" → 向量 A
文本："一只猫在沙发上" → 向量 B
余弦相似度：cos(A, B) = 0.95（高）

图片："一只猫在沙发上" → 向量 A
文本："一辆汽车在高速上" → 向量 C
余弦相似度：cos(A, C) = 0.12（低）
```

**推理过程**：
```
用户上传图片
↓
OpenCV解码（BGR → RGB）
↓
预处理（resize 224×224，归一化）
↓
CLIP模型编码
↓
输出512维向量
```

#### 6.3.1.3 OpenCV图像预处理

**图像解码**：
```java
// Maven依赖
// <dependency>
//     <groupId>org.openpnp</groupId>
//     <artifactId>opencv</artifactId>
//     <version>4.9.0</version>
// </dependency>

public class ImagePreprocessor {
    static {
        System.loadLibrary(Core.NATIVE_LIBRARY_NAME);
    }

    public static Mat decodeImage(byte[] imageData) {
        return Imgcodecs.imdecode(new MatOfByte(imageData), Imgcodecs.IMREAD_COLOR);
    }

    public static Mat resizeImage(Mat image, int width, int height) {
        Mat resized = new Mat();
       Imgproc.resize(image, resized, new Size(width, height));
        return resized;
    }

    public static float[] normalizeImage(Mat image) {
        // 转为float类型，归一化到[0,1]
        Mat normalized = new Mat();
        image.convertTo(normalized, CvType.CV_32F, 1.0/255.0);
        
        // 转为数组（简化版）
        int total = (int) (normalized.total() * normalized.channels());
        float[] data = new float[total];
        normalized.get(0, 0, data);
        return data;
    }
}
```

**预处理流程图**：
```
原始图片（BGR） → 解码（IMREAD_COLOR）
     ↓
resize 224×224（Imgproc.resize）
     ↓
转换类型（CV_8U → CV_32F）
     ↓
归一化（÷255.0）
     ↓
CLIP输入（[C,H,W]格式）
```

### 6.3.2 语音输入处理（Whisper模型）

#### 6.3.2.1 概念定义

**Whisper**：OpenAI开源的语音识别模型，支持100+种语言。

**设计目标**：
- ✅ 语音→文字（端到端ASR）
- ✅ 多语言支持（中文/英文/日文等）
- ✅ 时长处理（最长30秒/片段）

#### 6.3.2.2 技术原理

**模型架构**：
```
输入：音频波形（16kHz，单声道）
↓
音频预处理（预emphasis，梅尔频谱图）
↓
Encoder（Transformer，32层）
↓
Decoder（Transformer，32层）
↓
输出：文字序列（token）
```

**推理过程**：
```
用户语音输入（10秒）
↓
pydub格式转换（mp3 → wav）
↓
音频分段（30秒窗口，重叠1秒）
↓
Whisper模型编码
↓
输出文字（中文/英文）
```

#### 6.3.2.3 pydub音频预处理

**音频格式转换**：
```python
# Python脚本（调用Java主程序）
# pip install pydub openai

from pydub import AudioSegment
import openai

def convert_audio(input_path, output_path):
    """音频格式转换（mp3/wav/flac → wav）"""
    audio = AudioSegment.from_file(input_path)
    audio = audio.set_channels(1)  # 单声道
    audio = audio.set_frame_rate(16000)  # 16kHz
    audio.export(output_path, format="wav")
```

**音频分段策略**：
```
音频总长：60秒
窗口大小：30秒
重叠：1秒
分段数：2段（[0-30], [29-59]）
```

**Python代码**：
```python
def segment_audio(audio_path, segment_length=30, overlap=1):
    """音频分段（30秒窗口，重叠1秒）"""
    audio = AudioSegment.from_file(audio_path)
    segments = []
    
    for start in range(0, len(audio), segment_length * 1000 - overlap * 1000):
        end = start + segment_length * 1000
        segment = audio[start:end]
        segments.append(segment)
        
        if end >= len(audio):
            break
            
    return segments
```

### 6.3.4 常见误区

#### 6.3.4.1 误区1：图片越大越好

**错误**：认为高清图片（4K/8K）能提升CLIP精度。

**正确**：CLIP输入固定尺寸（224×224），高清图片需降采样，反而增加计算开销。

**建议**：
- 原始图片：不限制大小
- 预处理：统一resize 224×224
- 质量检查：过滤模糊/低 contrast图片

#### 6.3.4.2 误区2：语音越长越好

**错误**：认为超过30秒的语音能一次识别。

**正确**：Whisper单次处理上限30秒，长语音需分段+拼接。

**建议**：
- 音频分段：30秒窗口，重叠1秒
- 拼接策略：取重叠部分的平均（或投票）
- 后处理：拼写修正（Spell Check）

#### 6.3.4.3 误区3：多模态 = 简单拼接

**错误**：将图片向量 + 文本向量简单拼接（concat）。

**正确**：需要语义对齐（如：CLIP的联合嵌入空间）。

**建议**：
- 使用CLIP预训练模型（无需自己训练）
- 特征投影：全连接层映射到统一维度
- 注意力机制：Cross-Attention融合
