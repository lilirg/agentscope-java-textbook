## 生产环境部署

### 1. Docker容器化部署

**原理**：使用Docker容器化部署，提高部署效率与一致性。

**部署策略**：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **单容器部署** | 一个容器运行一个应用 | 小型应用 |
| **多容器部署** | 使用Docker Compose运行多个容器 | 中型应用 |
| **集群部署** | 使用Kubernetes管理容器集群 | 大型应用 |

**Docker优化**：
- **多阶段构建**：减小镜像体积
- **缓存层**：利用Docker缓存，加速构建
- **精简基础镜像**：使用`alpine`镜像

**Dockerfile示例**：
```dockerfile
# Dockerfile（多阶段构建）
# 第一阶段：构建
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /build
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests

# 第二阶段：运行
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=builder /build/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

---

### 2. Kubernetes集群部署

**原理**：使用Kubernetes管理容器集群，实现高可用与弹性伸缩。

**部署策略**：

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **Deployment** | 管理无状态应用 | Web应用 |
| **StatefulSet** | 管理有状态应用 | 数据库 |
| **DaemonSet** | 每个节点运行一个Pod | 日志收集 |

**K8s配置示例**：
```yaml
# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: agent-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: agent-app
  template:
    metadata:
      labels:
        app: agent-app
    spec:
      containers:
      - name: agent-app
        image: agent-app:latest
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "256Mi"
            cpu: "100m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /actuator/health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 5
        readinessProbe:
          httpGet:
            path: /actuator/ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 3
```

## 监控系统搭建

### 1. Prometheus监控

**原理**：Prometheus是开源监控系统，支持时序数据存储与查询。

**监控指标**：

| 指标 | 说明 | 优化目标 |
|------|------|---------|
| **CPU使用率** | CPU占用率 | <70% |
| **内存使用** | 内存占用 | <80% |
| **响应时间** | 从请求到响应的时间 | <1s |
| **API错误率** | API调用错误率 | <1% |
| **Pod状态** | Pod运行状态 | Running |

**指标收集**（Spring Boot Actuator）：
```yaml
# application.yml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
  metrics:
    tags:
      application: ${spring.application.name}
```

---

### 2. Grafana可视化

**原理**：Grafana是开源可视化工具，支持多种数据源。

**仪表盘配置**：

| 配置项 | 说明 |
|--------|------|
| **数据源** | Prometheus |
| **面板类型** | Graph、Gauge、Table |
| **查询语言** | PromQL |

**Grafana仪表盘示例**：
```
1. 添加数据源：Prometheus
2. 创建仪表盘：
   - CPU使用率（Gauge）
   - 内存使用（Graph）
   - 响应时间（Graph）
   - API错误率（Gauge）
3. 导入官方仪表盘：ID: 1860
```

## 告警系统配置

### 1. Prometheus AlertManager

**原理**：AlertManager是Prometheus的告警管理组件。

**告警规则**：
```yaml
# alertmanager.yml
groups:
  - name: agent-app-alerts
    rules:
      - alert: HighCPUUsage
        expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High CPU usage on {{ $labels.instance }}"
          description: "CPU usage is above 80% for more than 5 minutes."

      - alert: HighMemoryUsage
        expr: (node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes * 100 > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High memory usage on {{ $labels.instance }}"
          description: "Memory usage is above 80% for more than 5 minutes."
```

## 日志分析

### 1. ELK（Elasticsearch + Logstash + Kibana）

**原理**：ELK是日志分析解决方案，支持日志收集、存储、检索与可视化。

**架构**：
```
应用日志 → Logstash → Elasticsearch → Kibana
```

**日志格式**：
```json
{
  "timestamp": "2026-04-17 19:30:45",
  "level": "ERROR",
  "class": "AgentService",
  "message": "API调用超时"
}
```

**日志索引**：
```json
# Elasticsearch索引映射
{
  "mappings": {
    "properties": {
      "timestamp": { "type": "date" },
      "level": { "type": "keyword" },
      "class": { "type": "keyword" },
      "message": { "type": "text" }
    }
  }
}
```

---

### 2. Loki（轻量级日志系统）

**原理**：Loki是轻量级日志聚合系统，与Prometheus集成。

**架构**：
```
应用日志 → Promtail → Loki → Grafana
```

**日志标签**：
```yaml
# Promtail配置
scrape_configs:
  - job_name: application
    static_configs:
      - targets:
          - localhost
        labels:
          job: app
          __path__: /var/log/app/*.log
```
