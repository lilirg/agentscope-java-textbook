## 3.3 核心概念

> **本节学习目标**：掌握API接入开发的核心原理

---

### 3.3.1 API基础

#### 3.3.1.1 API定义

**API（Application Programming Interface）** 是应用程序之间通信的接口。

```
API = 接口定义 + 请求/响应
```

**API核心特性**：
| 特性 | 说明 |
|------|------|
| **接口定义** | 定义应用程序间的交互方式 |
| **数据交换** | 实现数据的传输与交换 |
| **功能调用** | 实现功能的调用与复用 |

#### 3.3.1.2 HTTP协议

**HTTP协议** 是API通信的基础协议。

```
HTTP协议 = 请求 + 响应 + 状态码
```

**HTTP协议核心概念**：
| 概念 | 说明 |
|------|------|
| **请求** | 客户端发送请求到服务器 |
| **响应** | 服务器返回响应到客户端 |
| **状态码** | 表示请求处理结果（200成功、400错误、500服务器错误） |

#### 3.3.1.3 请求/响应格式

| 格式 | 说明 |
|------|------|
| **请求格式** | URL + HTTP方法 + 请求头 + 请求体 |
| **响应格式** | 状态码 + 响应头 + 响应体 |

---

### 3.3.2 RESTful API

#### 3.3.2.1 RESTful API定义

**RESTful API** 是基于REST架构的API设计风格。

```
RESTful API = 资源 + HTTP方法 + 状态码
```

**RESTful API核心概念**：
| 概念 | 说明 |
|------|------|
| **资源** | 通过URL标识的实体（如/users、/posts） |
| **HTTP方法** | GET（查询）、POST（创建）、PUT（更新）、DELETE（删除） |
| **状态码** | 200（成功）、400（错误）、500（服务器错误） |

---

### 3.3.3 API文档

#### 3.3.3.1 API文档定义

**API文档** 是API的说明书，描述API的使用方式。

```
API文档 = 接口定义 + 请求/响应示例 + 错误码说明
```

**API文档核心概念**：
| 概念 | 说明 |
|------|------|
| **接口定义** | API的URL、HTTP方法、参数、响应 |
| **请求/响应示例** | API调用示例 |
| **错误码说明** | API错误码说明 |

---

### 3.3.4 API安全

#### 3.3.4.1 API安全定义

**API安全** 是保护API免受攻击的技术。

```
API安全 = 认证 + 授权 + 加密 + 防护
```

**API安全核心概念**：
| 概念 | 说明 |
|------|------|
| **认证** | 验证用户身份（API Key、OAuth2） |
| **授权** | 控制用户权限（RBAC、ABAC） |
| **加密** | 加密通信（HTTPS） |
| **防护** | 防止攻击（限流、防刷） |

---

### 3.3.5 API测试

#### 3.3.5.1 API测试定义

**API测试** 是验证API功能与性能的测试。

```
API测试 = 单元测试 + 集成测试 + 性能测试
```

**API测试核心概念**：
| 概念 | 说明 |
|------|------|
| **单元测试** | 测试单个API功能 |
| **集成测试** | 测试API与其他模块的集成 |
| **性能测试** | 测试API的性能指标 |

---

| 核心概念 | 实现要点 |
