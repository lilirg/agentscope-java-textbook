## Agent性能评测相关陷阱

### 陷阱1：JMH基准测试误差

**现象**：
```
Warning: JMH benchmark results are inconsistent (high variance)
```

**原因**：
- JVM预热不足
- GC干扰
- 系统负载波动

**调试方法**：
```java
// 1. 增加预热轮数
@Warmup(iterations = 5, time = 1, timeUnit = TimeUnit.SECONDS)

// 2. 增加测量轮数
@Measurement(iterations = 10, time = 1, timeUnit = TimeUnit.SECONDS)

// 3. 使用 fork 避免 JVM 优化干扰
@Fork(3)
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **增加预热** | 预热JVM，减少 JIT 编译影响 | 高精度测试 |
| **避免GC干扰** | 使用 `-XX:+UseG1GC` 参数 | 内存敏感测试 |
| **多次运行** | 取平均值，减少波动影响 | 通用 |

**示例**：
```java
@State(Scope.Thread)
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MILLISECONDS)
@Warmup(iterations = 5, time = 1, timeUnit = TimeUnit.SECONDS)
@Measurement(iterations = 10, time = 1, timeUnit = TimeUnit.SECONDS)
@Fork(3)
public class AgentBenchmark {

    @Benchmark
    public String processQuery() {
        return "Result";
    }
}
```

## 提示词优化相关陷阱

### 陷阱2：缓存未命中

**现象**：
```
Warning: Cache miss rate is too high (>50%)
```

**原因**：
- 缓存键设计不合理
- 缓存过期时间过短
- 缓存容量不足

**调试方法**：
```java
// 1. 检查缓存命中率
Cache<String, String> cache = Caffeine.newBuilder()
    .maximumSize(10_000)
    .recordStats()
    .build();

// 2. 获取命中率统计
CacheStats stats = cache.stats();
System.out.println("Hit rate: " + stats.hitRate());
System.out.println("Miss rate: " + stats.missRate());
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **优化缓存键** | 使用统一的缓存键生成策略 | 通用 |
| **调整过期时间** | 根据业务需求设置合理的TTL | 动态数据 |
| **增加缓存容量** | 增加 maximumSize | 大量数据 |

**示例**：
```java
public class CacheManager {
    private final Cache<String, String> cache = Caffeine.newBuilder()
        .maximumSize(100_000) // ✅ 增加缓存容量
        .expireAfterWrite(30, TimeUnit.MINUTES) // ✅ 延长TTL
        .recordStats() // ✅ 记录统计信息
        .build();

    public String get(String key) {
        return cache.getIfPresent(key);
    }

    public void put(String key, String value) {
        cache.put(key, value);
    }

    public CacheStats getStats() {
        return cache.stats();
    }
}
```

## API调用优化相关陷阱

### 陷阱3：批量处理超时

**现象**：
```
Error: Batch processing timeout (1000 queries processed in 30s)
```

**原因**：
- 批量处理过大（超过超时时间）
- 网络延迟
- API限流

**调试方法**：
```java
// 1. 检查分组大小
List<List<String>> groups = Lists.partition(queries, 10); // ✅ 减小分组
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **减小分组** | 减少每组查询数量 | 高延迟场景 |
| **异步处理** | 使用异步API调用 | 高并发场景 |
| **限流控制** | 控制请求速率 | API限流场景 |

**示例**：
```java
public class BatchAPI {
    private static final int BATCH_SIZE = 5; // ✅ 减小分组
    private static final int MAX_RETRIES = 3; // ✅ 增加重试次数

    public List<String> processBatchQueries(List<String> queries) {
        List<List<String>> groups = Lists.partition(queries, BATCH_SIZE);

        List<String> results = new ArrayList<>();
        for (List<String> group : groups) {
            for (int i = 0; i < MAX_RETRIES; i++) {
                try {
                    results.addAll(apiClient.batchProcess(group));
                    break; // 成功后跳出重试循环
                } catch (TimeoutException e) {
                    if (i == MAX_RETRIES - 1) {
                        throw e; // 重试次数用尽，抛出异常
                    }
                }
            }
        }

        return results;
    }
}
```

## Agent部署优化相关陷阱

### 陷阱4：Docker镜像过大

**现象**：
```
Warning: Docker image size is too large (>500MB)
```

**原因**：
- 依赖库过多
- 未使用多阶段构建
- 包含调试文件

**调试方法**：
```bash
# 1. 检查镜像大小
docker images

# 2. 检查镜像层
docker history agent-app:latest
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **多阶段构建** | 使用多阶段构建减小镜像体积 | 通用 |
| **精简基础镜像** | 使用`alpine`镜像 | 生产环境 |
| **排除调试文件** | 移除调试依赖 | 生产环境 |

**示例**：
```dockerfile
# Dockerfile（多阶段构建）
# 第一阶段：构建
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /build
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests

# 第二阶段：运行
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY --from=builder /build/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 日志增强（SLF4J）

**错误示例**：
```java
System.out.println("Error: " + e.getMessage());
```

**正确示例**：
```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class PerformanceMonitor {
    private static final Logger logger = LoggerFactory.getLogger(PerformanceMonitor.class);
    
    public void monitorResponseTime(long responseTime) {
        logger.debug("Monitoring response time: {}ms", responseTime);
        
        if (responseTime > 1000) {
            logger.error("响应时间超限: {}ms", responseTime);
            sendAlert("响应时间超限", "响应时间：" + responseTime + "ms");
        }
    }
}
```
