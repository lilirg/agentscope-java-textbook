### 10.5.1 性能瓶颈陷阱

#### 陷阱1：LLM推理速度慢

**问题描述**：
```
问题：LLM推理速度慢，响应时间长
症状：
- API响应时间长（>5秒）
- QPS低
- 用户等待时间长

原因：
- 模型参数量大
- 批处理大小小
- 未启用GPU加速
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **模型量化** | 使用INT8/FP16量化模型 | GPU资源有限场景 |
| **批处理优化** | 增大批处理大小 | 高并发场景 |
| **模型蒸馏** | 使用轻量级蒸馏模型 | 资源受限场景 |

**示例代码**：
```java
// 模型量化
public class QuantizedLLM {
    private final Pipeline pipeline;
    
    public QuantizedLLM() {
        // 使用量化模型
        pipeline = new Pipeline("distilgpt2"); // 蒸馏模型
        pipeline.enableQuantization(); // 启用INT8量化
    }
    
    public String generate(String prompt) {
        return pipeline.generate(prompt);
    }
}

// 批处理优化
public class BatchInferenceService {
    private final LLMService llmService;
    private static final int BATCH_SIZE = 32;
    
    public List<String> generateBatch(List<String> prompts) {
        List<String> results = new ArrayList<>();
        
        for (int i = 0; i < prompts.size(); i += BATCH_SIZE) {
            List<String> batch = prompts.subList(i, 
                Math.min(i + BATCH_SIZE, prompts.size()));
            
            // 批量推理
            results.addAll(llmService.generate(batch));
        }
        
        return results;
    }
}

// 模型蒸馏
public class DistilledLLM {
    private final Pipeline pipeline;
    
    public DistilledLLM() {
        // 使用蒸馏模型（如distilgpt2）
        pipeline = new Pipeline("distilgpt2");
    }
    
    public String generate(String prompt) {
        return pipeline.generate(prompt);
    }
}
```

#### 陷阱2：数据库查询慢

**问题描述**：
```
问题：数据库查询速度慢，影响整体性能
症状：
- 数据库响应时间长（>1秒）
- SQL执行计划不佳
- 索引缺失

原因：
- 未创建索引
- SQL未优化
- 数据量过大
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **索引优化** | 创建复合索引 | 高频查询字段 |
| **SQL优化** | 使用EXPLAIN分析执行计划 | 慢查询场景 |
| **分页查询** | 避免全表扫描 | 大数据量场景 |

**示例代码**：
```java
// 索引优化
@Configuration
public class DatabaseConfig {
    @Bean
    public DatabaseInitializer databaseInitializer(JdbcTemplate jdbcTemplate) {
        return new DatabaseInitializer(jdbcTemplate);
    }
}

public class DatabaseInitializer {
    private final JdbcTemplate jdbcTemplate;
    
    public DatabaseInitializer(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }
    
    @PostConstruct
    public void initialize() {
        // 创建索引
        jdbcTemplate.execute("CREATE INDEX IF NOT EXISTS idx_messages_user_id ON messages(user_id)");
        jdbcTemplate.execute("CREATE INDEX IF NOT EXISTS idx_messages_created_at ON messages(created_at)");
        jdbcTemplate.execute("CREATE INDEX IF NOT EXISTS idx_users_email ON users(email)");
        
        // 创建复合索引
        jdbcTemplate.execute("CREATE INDEX IF NOT EXISTS idx_messages_user_created ON messages(user_id, created_at)");
    }
}

// SQL优化
public class OptimizedQueryService {
    private final JdbcTemplate jdbcTemplate;
    
    public List<Message> findMessagesByUserId(String userId, int offset, int limit) {
        String sql = "SELECT * FROM messages WHERE user_id = ? ORDER BY created_at DESC LIMIT ? OFFSET ?";
        return jdbcTemplate.query(sql, new Object[]{userId, limit, offset}, 
            new BeanPropertyRowMapper<>(Message.class));
    }
    
    public List<Message> findRecentMessages(String userId, int limit) {
        String sql = "SELECT * FROM messages WHERE user_id = ? ORDER BY created_at DESC LIMIT ?";
        return jdbcTemplate.query(sql, new Object[]{userId, limit}, 
            new BeanPropertyRowMapper<>(Message.class));
    }
}

// 分页查询
public class PaginationService {
    private final JdbcTemplate jdbcTemplate;
    
    public Page<Message> findMessagesPage(String userId, int page, int size) {
        int offset = page * size;
        String sql = "SELECT * FROM messages WHERE user_id = ? ORDER BY created_at DESC LIMIT ? OFFSET ?";
        
        List<Message> messages = jdbcTemplate.query(sql, new Object[]{userId, size, offset}, 
            new BeanPropertyRowMapper<>(Message.class));
        
        // 查询总数
        String countSql = "SELECT COUNT(*) FROM messages WHERE user_id = ?";
        int total = jdbcTemplate.queryForObject(countSql, new Object[]{userId}, Integer.class);
        
        return new Page<>(messages, page, size, total);
    }
}
```

### 10.5.2 监控陷阱

#### 陷阱3：性能监控指标缺失

**问题描述**：
```
问题：性能监控缺乏关键指标，无法及时发现问题
症状：
- 性能问题发现滞后
- 无法定位瓶颈
- 缺乏性能基线

原因：
- 未记录关键指标
- 未设置告警阈值
- 未进行性能基准测试
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **关键指标记录** | 记录API响应时间、QPS、错误率 | 通用场景 |
| **告警阈值设置** | 设置告警阈值（如响应时间>5秒） | 生产环境 |
| **性能基准测试** | 定期进行性能基准测试 | 性能调优场景 |

**示例代码**：
```java
// 关键指标记录
public class PerformanceMetrics {
    private final Map<String, List<Long>> responseTimes = new ConcurrentHashMap<>();
    private final AtomicLong totalRequests = new AtomicLong(0);
    private final AtomicLong errorRequests = new AtomicLong(0);
    
    public void recordResponseTime(String endpoint, long duration) {
        responseTimes.computeIfAbsent(endpoint, k -> new ArrayList<>()).add(duration);
        totalRequests.incrementAndGet();
    }
    
    public void recordError() {
        errorRequests.incrementAndGet();
    }
    
    public double getAverageResponseTime(String endpoint) {
        List<Long> times = responseTimes.getOrDefault(endpoint, new ArrayList<>());
        if (times.isEmpty()) {
            return 0;
        }
        return times.stream().mapToLong(Long::longValue).average().orElse(0);
    }
    
    public double getErrorRate() {
        long total = totalRequests.get();
        if (total == 0) {
            return 0;
        }
        return (double) errorRequests.get() / total;
    }
}

// 告警阈值设置
public class PerformanceAlerting {
    private static final long RESPONSE_TIME_THRESHOLD = 5000; // 5秒
    private static final double ERROR_RATE_THRESHOLD = 0.05; // 5%
    
    private final PerformanceMetrics metrics;
    
    public PerformanceAlerting(PerformanceMetrics metrics) {
        this.metrics = metrics;
    }
    
    public void checkAlerts() {
        // 检查响应时间
        double avgResponseTime = metrics.getAverageResponseTime("/api/generate");
        if (avgResponseTime > RESPONSE_TIME_THRESHOLD) {
            logger.warn("Response time exceeded threshold: {}ms > {}ms", 
                avgResponseTime, RESPONSE_TIME_THRESHOLD);
        }
        
        // 检查错误率
        double errorRate = metrics.getErrorRate();
        if (errorRate > ERROR_RATE_THRESHOLD) {
            logger.warn("Error rate exceeded threshold: {}% > {}%", 
                errorRate * 100, ERROR_RATE_THRESHOLD * 100);
        }
    }
}

// 性能基准测试
public class PerformanceBenchmark {
    private final LLMService llmService;
    
    public void benchmark() {
        // 准备测试数据
        List<String> prompts = IntStream.range(0, 1000)
            .mapToObj(i -> "Test prompt " + i)
            .collect(Collectors.toList());
        
        // 测试单次推理
        long singleStart = System.currentTimeMillis();
        llmService.generate("Test prompt");
        long singleDuration = System.currentTimeMillis() - singleStart;
        
        // 测试批量推理
        long batchStart = System.currentTimeMillis();
        llmService.generate(prompts);
        long batchDuration = System.currentTimeMillis() - batchStart;
        
        logger.info("Performance benchmark results:");
        logger.info("Single inference: {}ms", singleDuration);
        logger.info("Batch inference (1000 prompts): {}ms", batchDuration);
        logger.info("Throughput: {} prompts/s", 1000.0 / (batchDuration / 1000.0));
    }
}
```

#### 陷阱4：内存泄漏

**问题描述**：
```
问题：AI应用存在内存泄漏，长期运行后内存溢出
症状：
- JVM内存持续增长
- GC频率增加
- OutOfMemoryError

原因：
- 未关闭资源（如数据库连接）
- 缓存未清理
- 静态集合无限增长
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **资源管理** | 使用try-with-resources | 通用场景 |
| **缓存清理** | 定期清理缓存 | 缓存场景 |
| **WeakReference** | 使用WeakReference存储缓存 | 大对象场景 |

**示例代码**：

### 10.5.3 优化陷阱

#### 陷阱5：过度优化导致代码复杂

**问题描述**：
```
问题：过度追求性能优化导致代码复杂、可维护性差
症状：
- 代码难以理解
- 难以调试
- 维护成本高

原因：
- 提前优化（未找到瓶颈）
- 优化手段过于激进
- 未权衡性能与可维护性
```

**解决方案**：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| **性能 profiling** | 先 profiling 再优化 | 通用场景 |
| **渐进式优化** | 逐步优化，每次验证效果 | 高安全场景 |
| **文档说明** | 记录优化原因与效果 | 复杂优化场景 |

**示例代码**：
```java
// 性能 profiling
public class PerformanceProfiler {
    private final Map<String, Long> timings = new ConcurrentHashMap<>();

        long start = System.currentTimeMillis();
        T result = action.get();
        long duration = System.currentTimeMillis() - start;
        
        timings.compute(name, (k, v) -> v != null ? v + duration : duration);
        
        return result;

        logger.info("Performance profiling report:");
        timings.forEach((name, duration) ->
```
