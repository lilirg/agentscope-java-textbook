### 10.3.1 Agent性能评测

#### 1. JMH基准测试

**原理**：JMH是Java微基准测试框架，用于精确测量代码性能。

**测试类型**：
- **AverageTime**：平均响应时间（越低越好）
- **Throughput**：吞吐量（越高越好）
- **SampleTime**：采样响应时间（95%分位）

**配置示例**：
```java
@State(Scope.Thread)
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MILLISECONDS)
public class AgentBenchmark {

    @Benchmark
    public String processQuery() {
        // 模拟Agent处理查询
        return "Result";
    }
}
```

**运行结果**：
```
Benchmark                       Mode  Cnt   Score   Error  Units
AgentBenchmark.processQuery    avgt   30  12.456 ± 0.234  ms/op
```

#### 2. 性能指标

| 指标 | 说明 | 优化目标 |
|------|------|---------|
| **响应时间** | 从请求到响应的时间 | <1s |
| **吞吐量** | 单位时间处理请求数 | >100 QPS |
| **CPU使用率** | CPU占用率 | <70% |
| **内存使用** | 内存占用 | <2GB |

### 10.3.2 提示词优化

#### 1. Token消耗优化

**原理**：提示词越短，Token消耗越少，响应速度越快，成本越低。

**优化策略**：

| 策略 | 说明 | 效果 |
|------|------|------|
| **精简提示词** | 去除冗余描述 | 减少30% Token |
| **使用占位符** | 动态填充变量 | 减少20% Token |
| **分段处理** | 分阶段生成提示词 | 减少10% Token |

**优化示例**：
```java
// ❌ 优化前
String prompt = "你是一个AI助手，你的目标是回答用户的问题。请回答以下问题：" + query;

// ✅ 优化后
String prompt = "{query}"; // 使用占位符
```

---

#### 2. 缓存策略

**原理**：缓存重复请求结果，避免重复调用API。

**缓存类型**：
- **本地缓存**：Caffeine（高性能）
- **分布式缓存**：Redis（跨节点）

**缓存策略**：
- **读取**：先查缓存，缓存未命中再调用API
- **写入**：API返回后写入缓存
- **过期**：设置TTL（Time To Live），自动过期

**缓存示例**：
```java
@Service
public class AgentService {
    private final Cache<String, String> cache = Caffeine.newBuilder()
        .maximumSize(10_000)
        .expireAfterWrite(10, TimeUnit.MINUTES)
        .build();

    public String processQuery(String query) {
        // 1. 查缓存
        String cached = cache.getIfPresent(query);
        if (cached != null) {
            return cached;
        }

        // 2. 调用API
        String result = apiClient.call(query);

        // 3. 写缓存
        cache.put(query, result);

        return result;
    }
}
```

### 10.3.3 API调用优化

#### 1. 批量处理

**原理**：批量处理多个请求，减少API调用次数。

**优化策略**：
- **分组**：将多个小请求合并为批量请求
- **异步**：使用异步API调用，提高并发

**批量处理示例**：
```java
public class BatchAPI {
    private final APIClient apiClient;

    public List<String> processBatchQueries(List<String> queries) {
        // 1. 分组（每组10个查询）
        List<List<String>> groups = Lists.partition(queries, 10);

        // 2. 批量处理
        List<String> results = new ArrayList<>();
        for (List<String> group : groups) {
            results.addAll(apiClient.batchProcess(group));
        }

        return results;
    }
}
```

---

#### 2. 异步调用

**原理**：使用异步API调用，提高并发处理能力。

**异步调用示例**：
```java
@Service
public class AsyncAgentService {
    private final APIClient apiClient;

    public CompletableFuture<String> processQueryAsync(String query) {
        return CompletableFuture.supplyAsync(() -> {
            return apiClient.call(query);
        });
    }

    public CompletableFuture<List<String>> processBatchQueriesAsync(List<String> queries) {
        List<CompletableFuture<String>> futures = queries.stream()
            .map(query -> CompletableFuture.supplyAsync(() -> apiClient.call(query)))
            .collect(Collectors.toList());

        return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenApply(v -> futures.stream()
                .map(CompletableFuture::join)
                .collect(Collectors.toList()));
    }
}
```

### 10.3.4 Agent部署优化

#### 1. 容器化部署

**原理**：使用Docker容器化部署，提高部署效率与一致性。

**Docker优化**：
- **多阶段构建**：减小镜像体积
- **缓存层**：利用Docker缓存，加速构建
- **精简基础镜像**：使用`alpine`镜像

**Dockerfile优化**：
```dockerfile
# 优化前
FROM openjdk:17
COPY target/app.jar app.jar
ENTRYPOINT ["java", "-jar", "app.jar"]

# 优化后（多阶段构建）
FROM openjdk:17-jdk-slim AS runtime
WORKDIR /app
COPY --from=builder /app/target/app.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

---

#### 2. 负载均衡

**原理**：使用负载均衡，分配流量到多个Agent实例。

**负载均衡策略**：
- **轮询**：按顺序分配流量
- **加权轮询**：按权重分配流量
- **最小连接**：分配到连接最少的实例

**Nginx配置示例**：
```nginx
# nginx.conf
upstream agent_cluster {
    least_conn;  # 最小连接策略
    server agent1:8080 weight=3;
    server agent2:8080 weight=2;
    server agent3:8080 weight=1;
}

server {
    listen 80;
    location /api/ {
        proxy_pass http://agent_cluster;
    }
}
```

### 10.3.5 性能监控

#### 1. 日志监控

**原理**：记录性能日志，分析性能瓶颈。

**日志内容**：
- API调用时间
- 缓存命中率
- CPU/内存使用率

**日志配置**：
```xml
<!-- logback.xml -->
<appender name="PERF" class="ch.qos.logback.core.rolling.RollingFileAppender">
    <file>logs/performance.log</file>
    <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
        <fileNamePattern>logs/performance.%d{yyyy-MM-dd}.log</fileNamePattern>
    </rollingPolicy>
    <encoder>
        <pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
    </encoder>
</appender>

<logger name="performance" level="INFO" additivity="false">
    <appender-ref ref="PERF"/>
</logger>
```

---

#### 2. 监控告警

**原理**：监控性能指标，超阈值时告警。

**监控指标**：
- 响应时间 >1s 告警
- CPU使用率 >80% 告警
- 内存使用率 >80% 告警

**告警配置**：
```java
@Service
public class PerformanceMonitor {
    private static final Logger logger = LoggerFactory.getLogger("performance");

    public void monitorResponseTime(long responseTime) {
        if (responseTime > 1000) { // 超过1秒
            logger.error("响应时间超限: {}ms", responseTime);
            sendAlert("响应时间超限", "响应时间：" + responseTime + "ms");
        }
    }

    private void sendAlert(String title, String message) {
        // 钉钉告警
        DingTalkNotifier.notify(title, message);
    }
}
```
