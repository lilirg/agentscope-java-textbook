
# 附录A：核心概念速查表

> 本附录提供Agent开发相关核心概念的快速参考，方便开发过程中查阅

## A.1 Agent核心概念

| 概念 | 定义 | 关键要点 |
|------|------|---------|
| **Agent（智能体）** | 能够自主执行任务、做出决策、与环境交互的人工智能系统 | - 自主性<br>- 感知能力<br>- 规划能力<br>- 学习能力<br>- 交互能力 |
| **LLM（大语言模型）** | Agent的"大脑"，负责理解意图、逻辑判断、生成文本 | - GPT-4、Claude、DeepSeek、通义千问<br>- 无法直接执行外部操作<br>- 需配合Tool使用 |
| **Planner（规划器）** | 把大目标拆解成小步骤 | - 提示词链<br>- 决策树<br>- 图规划 |
| **Tool Executor（工具执行器）** | 调用外部能力（搜索、计算、API） | - REST客户端<br>- 数据库驱动<br>- API SDK |
| **Memory（记忆系统）** | 保存对话历史和用户偏好 | - 短期记忆（LinkedList）<br>- 长期记忆（JSON + Redis）<br>- 上下文记忆 |
| **Single Agent** | 单个Agent独立工作 | - 适用于简单任务<br>- 实现简单<br>- 扩展性有限 |
| **Multi-Agent** | 多个Agent协作工作 | - 适用于复杂任务<br>- 实现复杂<br>- 扩展性强 |

## A.2 Memory系统

| 概念 | 定义 | 关键要点 |
|------|------|---------|
| **短期记忆** | Agent在当前对话窗口内保留的历史消息 | - 数据结构：LinkedList<br>- 窗口策略：固定窗口（默认20条）<br>- 问题：上下文溢出 |
| **长期记忆** | Agent跨对话保存的用户偏好、历史记录与私有知识 | - 数据格式：JSON<br>- 缓存策略：Redis（毫秒级）+ 文件（断电不丢失）<br>- Key设计：user:{userId} |
| **RAG（检索增强生成）** | 在生成响应前，先从私有知识库检索相关信息，再喂给LLM生成答案 | - 核心步骤：向量化 → 检索 → 生成<br>- 向量化模型：Sentence Transformer<br>- 向量数据库：Chroma / Pinecone |
| **向量化** | 将文本转换为向量表示 | - 模型：all-MiniLM-L6-v2<br>- 维度：384维<br>- 算法：余弦相似度 |
| **向量相似度** | 计算两个向量的相似程度 | - 算法：余弦相似度<br>- 取值范围：-1到1<br>- 语义相似度匹配 |
